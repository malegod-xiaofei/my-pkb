### Neo4j图算法概况
#### 背景
+ 在很多真实的业务场景中，我们会选择图的模式来存储和检索数据，比如：社交网络，欺诈网络，网页排名，交通路径等等；这时我们会选择一些图数据库帮助我们进行存储和检索工作，neo4j是近年来比较成熟稳定的图数据库选择。社区版适用于中等业务场景规模和离线检索。
+ 当我们选定了这样的数据库，也就是选择了它的相关生态，neo4j生态发展至今，除了存储和检索功能外，还给我我们提供了非常多的图算法，以便于我们能够直接进行一些利于深度学习的工作，比如找到一些有用的基于图的特征规则（N度关系，这在传统表格数据库中查询起来很困难），比如做基于图的用户Embedding，再比如在图中进行最佳节点排名等等。
+ 我们可以让模型利用这些“特征” 进行更好的预测，下面我们就来看一看neo4j自带的图算法工具（neo4j-4.0.0版本以上），这个工具的名字叫做Neo4j Graph Data Science，简称GDS，在这里，我们将讲解GDS的使用，包括它的语法，算法的输入，输出和一般的使用场景，这是相对简单的部分（因为它封装了算法）。同时，我们也将结合论文，将这些算法做通俗易懂的阐述和解释。

#### 当前包括的算法
+ 中心性算法（节点排序算法）
+ 社区检测算法（找到指定模式的图）
+ 相似性算法（对比两个图的相似度）
+ 路径查找算法（找到图中的最佳路径）
+ 关系预测算法（找出/预测图中重要的关系）
+ 节点嵌入算法（对图中节点进行张量表示）

---

#### 节点嵌入算法
+ 我们希望通过图找到一些“特征”用于深度模型训练或者直接的用户推荐比较，因为对于graph embedding也就是`节点嵌入算法`我们将深入研究。
+ 节点嵌入算法（对图中节点进行张量表示）
    - Node2Vec 
    - GraphSAGE （图聚合）
    - Random Projection（随机投影）(FastRP)

---

+ 上述三种算法的原理和应用将详细介绍。在介绍之前，我们先聊一聊不同graph embedding的思想（用指定维度的向量表示节点）。
    - 降维思想：无论你是否学习过图论，你都应该知道在图中，一个节点的表示可以使用`邻接矩阵`，设图中存在100个节点，那么每个节点的表示都是100x100的矩阵，矩阵是稀疏，它的值由0/1构成，代表是否存在着边。那么如果我想用20维的向量表示节点，那么我需要对100x100的矩阵降维。这是随机投影的思想。
    - 序列思想：我们知道节点与节点通过边（关系）来连接，这些连接的节点某种程度上代表着序列，从左到右或者从右到左无所谓，我们知道他们都是合理的，但是绝对不应该跳跃，这很像一种文本表达。因此，我们可以通过这些序列来训练word2vec，那么怎么得到这些序列呢，random walk是个非常好的方法。这就是Node2Vec的思想。
    - 动态思想：上述的思想某个节点的embedding都被认为是一组固定数值，但是这对于新节点的表示非常不友好，因为新节点要融入图中，就会产生新的边，这些边又连接在老得节点上，那么老节点的embedding就应该变化，没错，新节点的表示带来巨大的改变和代价。因此我们不希望embedding是固定的，我们希望他应该是一个或一组聚合函数，这个函数带有参数，可能是我们熟悉的y = kx或者一个神经网络LSTM。

### Node2Vec
+ 如果你学过word2vec，那么你将会很容易理解它，它是random walk（随机游走）+ word2vec完成的节点embedding方式。
+ 下面我们将介绍random walk和word2vec。
    - random walk是一个有趣的算法，原理类似于著名的赌徒输光问题。在图中，假设从某一点出发，我们需要得到具有3阶关系的路径（1-2-3-4），当选择随机游走，达到下一个连接点的概率总是相同的。我们可以得到许多这样的路径，这些路径就好像一条条文本，就可以继续使用word2vec进行embedding。
    - word2vec是我们之前学过的可以一起回顾一下。
+ random walk应用代码：

```plain
CREATE (home:Page {name: 'Home'}),
       (about:Page {name: 'About'}),
       (product:Page {name: 'Product'}),
       (links:Page {name: 'Links'}),
       (a:Page {name: 'Site A'}),
       (b:Page {name: 'Site B'}),
       (c:Page {name: 'Site C'}),
       (d:Page {name: 'Site D'}),

       (home)-[:LINKS]->(about),
       (about)-[:LINKS]->(home),
       (product)-[:LINKS]->(home),
       (home)-[:LINKS]->(product),
       (links)-[:LINKS]->(home),
       (home)-[:LINKS]->(links),
       (links)-[:LINKS]->(a),
       (a)-[:LINKS]->(home),
       (links)-[:LINKS]->(b),
       (b)-[:LINKS]->(home),
       (links)-[:LINKS]->(c),
       (c)-[:LINKS]->(home),
       (links)-[:LINKS]->(d),
       (d)-[:LINKS]->(home)
```

```plain
MATCH (home:Page {name: 'Home'})
CALL gds.alpha.randomWalk.stream({
  nodeProjection: '*',
  relationshipProjection: {
    LINKS: {
      type: 'LINKS',
      orientation: 'UNDIRECTED'
    }
  },
  start: id(home),
  steps: 3,
  walks: 1
})
YIELD nodeIds
UNWIND nodeIds AS nodeId
RETURN gds.util.asNode(nodeId).name AS page
```

### Random Projection（随机投影）
+ 在数学上，这是一种降维的方法，类似于PCA或SVD，但是在超大规模的稀疏矩阵上其性能超过传统降维方法N个数量级。https://arxiv.org/pdf/1908.11512.pdf
+ 如果看论文非常枯燥，那么就用下面简单的方式理解一下，随机投影起源于高维向量的欧式距离（相似度）计算，比如文本的相似度计算，假如你当前的两个文本表示有10000维，那么传统的欧式距离计算就是（自己想），这对于60年代的计算力实在有一些难度，这时，我们需要一种近似的方式，假如我们能计算的维度上限是100，将10000到100维如何计算呢？我们只需要构造一个随机投影矩阵，1x10000 dot 10000x100 = 1x100，10000x100的矩阵就是随机投影矩阵，随机矩阵的第一行是从高斯分布上均匀选择的随机单位向量，第二行是来自与第一行正交的空间的随机单位向量，第三行是来自与前两行正交的空间的随机单位矢量，依此类推。（为什么呢，马尔可夫不等式，切比雪夫不等式，大数定理...）

### GraphSAGE（图聚合）
+ 正如之前所说，GraphSAGE是动态的embedding，论文：https://arxiv.org/pdf/1706.02216.pdf
+ 我通过原论文的中的一幅图来理解它的实现原理：

<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2024/png/25755350/1730543454946-011ffbc8-8e9b-4353-967e-79bf01eb7f4f.png)

+ 第一步：选定某个中心节点的一些邻居节点组成路径，怎么选呢，通过大量实验表明，K（关系的度）=2是个非常不错的选择，没错，这里还可以使用随机游走。
+ 第二步：从最远的邻居开始向中心聚合，如果K=2，那么就需要两种聚合函数，聚合函数可以使用一层MLP（就是一次矩阵运算+激活），它的参数随机初始化，输出就是预定义维度的图表征。我们需要更新它的参数，就需要损失函数，对于无监督学习，该方式非常类似UDAT（你还记得吗），我们希望距离更近的两个节点尽可能的相似。
+ 第三步：我们可以开始应用这个Embedding了。
