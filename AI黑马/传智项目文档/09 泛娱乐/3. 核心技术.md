## 核心技术
## 召回模块
---

#### 热门召回定义:
+ 取当前交互操作得分最多的前n个帖子入召回池

#### 时间召回定义:
+ 取最新发布的前n个帖子入召回池

#### 速度召回定义:
+ 取单位时间被进行交互操作次数最多的前n个帖子入召回池

#### 加速度召回定义:
+ 取单位时间内速度变化（上升）最快的前n个帖子入召回池

#### 协同过滤召回定义:
+ 取与用户隐含向量最相似的n个帖子

#### 明星二度关系召回定义:
+ 构建明星-帖子关系图谱，取二度关系明星相关的帖子

#### 自定义召回定义:
+ 由产品运营根据产品活动等自定义的召回策略

### 召回"金子塔"机制
因为考虑到计算资源在数据变动时的滞后性，以及如何满足个别用户快速消费的需求, 我们引入召回金字塔模型, 并为每一位用户维护一组该模型。

在所有的召回策略计算完成后，数据流除了直接走向规则过滤器外，同时会进入设计好的存储器，如：ES集群，

满足公共召回策略帖子构成"金字塔"的底层，满足个性化召回策略帖子构成"金字塔"的中层, 各项召回策略叠加的帖子处在上层，

从塔尖到塔底曝光比率逐渐增加。塔尖数据一般会优先进入下一个流程。 用户一次一般只会消费金子塔中的部分数据，因为全局召回按照窗口时间有序进行，

因此在此期间的用户推荐只需重新计算金字塔中的数据，并重新排列，保证金字塔数据的用户最快消费时间< 全局召回时间, 就可以保证用户获取到新的数据，并及时响应了用户最新行为。

### 规则过滤器（服务）
我们将为每一位用户维护一项规则过滤器，来保证固定间隔时间内不会重复推荐，并对同批不同用户上传的高重复率帖子进行比较去重, 合规审核等功能。

## 排序模块
---

### 用户标签体系设计
#### 用户静态标签
+ 用户的静态标签体系：包括用户昵称，年龄，星座，所持设备类型，价格，个性签名等。

#### 用户动态标签
+ 用户的动态标签体系: 最近浏览的明星，最近登陆时间，主动点赞次数，主动评论次数，主动转发次数等。

#### 挖掘类标签
+ 将用户的个性签名和历史评论内容使用标签化系统进行兴趣倾向预测，获得对应标签。

#### 业务关注类标签
+ 比如用户付费情况, 购买和消费情况。

### 帖子标签体系设计
#### 帖子静态标签
+ 帖子的静态标签体系：包括帖子关联明星，发布时间，title，帖子来源，发布的明星等。

#### 帖子动态标签
+ 用户的动态标签体系: 最近被浏览的次数，被点赞次数，被评论次数，被转发次数等。

#### 挖掘类标签
+ 将帖子title和评论内容使用标签化系统进行兴趣倾向预测，获得对应标签。

### 标签体系的计算与存储
标签体系按照定义进行计算，并每周进行一次更新，存储方式采用双层存储模式，Hbase将会记录所有历史标签数据，以便计算产生基于此的新标签。Neo4j会存储所有最新的标签数据，并负责将双标签体系融合。

对数据可视化和推荐的可解释性产生重要影响。

### 制定正负样本和训练集
+ 正样本定义: 用户对帖子进行了转发，评论，点赞操作，则该用户对应的画像特征+帖子对应的画像特征作为样本特征，该样本为正样本，用标签1表示。 
+ 负样本定义：用户对帖子进行了举报或标记不喜欢或在对其曝光多次后仍没有任何操作，则该用户对应的画像特征+帖子对应的画像特征作为样本特征，该样本为负样本，用标签0表示。

训练数据正负样本的采样比例：1:1, 以保证准确率随机基线为50%

### 特征工程
+ 由于采用树模型特点，特征需将“类别特征”进行multi-hot编码。
+ 可对特征进行规范化操作使其数值在指定区间变动，以加速模型收敛速度。

```plain
def fea_process(self,raw_fea):
        for fea in dummies_list:
            duplicate_df = raw_fea[fea].drop_duplicates()
            mapper = dict(zip(duplicate_df.values,
                     map(lambda x:[x], duplicate_df.index)))
            one_hot_encoder = OneHotEncoder().fit(list(mapper.values()))
            new_fea = one_hot_encoder.transform(list(map(lambda x: mapper[x],
                          raw_fea[fea].values)))
            raw_fea = pd.concat([raw_fea,
                          pd.DataFrame(new_fea.toarray())],
                          axis=1).drop([fea], axis =1)

            path = self.category_path+'/encoder@'+fea
            if not os.path.exists(path):os.makedirs(path)
            with open(path+'/mapper', 'w') as f:
                f.write(str(mapper))
            joblib.dump(one_hot_encoder, path+'/onehotencoder')
        return raw_fea.values
```

### 模型选择
考虑实际工程应用中初期样本数量有限，画像特征有限，总体特征为度不高时，将使用基于决策树的传统分类模型（gbdt+LR／randomforest）, 后期随着样本数量增大和特征维度的增加，将采用wide_deep模型。  
wide_deep模型理论请参考:[Wide & Deep Learning: Better Together with TensorFlow](https://ai.googleblog.com/2016/06/wide-deep-learning-better-together-with.html)
