## 1. 进程和线程有什么区别？

**答案要点：**

- 进程是资源分配的基本单位，拥有独立地址空间；线程是 CPU 调度的基本单位，共享所属进程的资源。
- 进程间通信成本较高，线程间共享数据更方便但需要同步控制。
- 进程崩溃通常不会直接影响其他进程；线程崩溃可能影响整个进程。
- Python 中 CPU 密集型任务更适合多进程，I/O 密集型任务更适合多线程或协程。

## 2. Python 中 GIL 是什么？它对多线程有什么影响？

**答案要点：**

GIL 是 CPython 解释器中的全局解释器锁，同一时刻通常只允许一个线程执行 Python 字节码。

- 对 CPU 密集型任务，多线程不能充分利用多核 CPU。
- 对 I/O 密集型任务，线程在等待 I/O 时会释放 GIL，因此多线程仍然有效。
- 如果需要并行计算，可以使用 `multiprocessing`、C 扩展、NumPy 向量化或分布式计算。

## 3. 线程安全问题是怎么产生的？如何解决？

**答案要点：**

多个线程同时读写共享变量，且操作不是原子的，就可能产生竞态条件。

常见解决方式：

- 使用互斥锁保护临界区。
- 使用线程安全队列传递数据。
- 减少共享状态，优先使用不可变对象。
- 使用线程池统一管理线程生命周期。

## 4. 什么是死锁？如何避免？

**答案要点：**

死锁是多个线程或进程互相等待对方持有的资源，导致所有参与者都无法继续执行。

常见避免方式：

- 固定加锁顺序。
- 减少锁粒度和持锁时间。
- 使用超时锁，避免无限等待。
- 避免在持锁期间执行阻塞 I/O 或调用复杂外部逻辑。

## 5. 进程池和线程池有什么作用？

**答案要点：**

池化的核心目的是复用工作单元，避免频繁创建和销毁进程/线程的开销。

- 进程池适合 CPU 密集型任务。
- 线程池适合 I/O 密集型任务。
- Python 中常用 `concurrent.futures.ProcessPoolExecutor` 和 `ThreadPoolExecutor`。
- 池大小需要根据 CPU 核数、I/O 阻塞比例、任务耗时综合调整。

## 6. TCP 客户端和服务端的基本开发流程是什么？

**答案要点：**

服务端：

1. 创建 socket。
2. 绑定 IP 和端口。
3. 监听连接。
4. 接收客户端连接。
5. 收发数据。
6. 关闭连接。

客户端：

1. 创建 socket。
2. 连接服务端。
3. 收发数据。
4. 关闭连接。

## 7. TCP 和 UDP 有什么区别？

**答案要点：**

- TCP 面向连接、可靠传输、有序传输、拥塞控制，适合文件传输、HTTP、数据库连接等。
- UDP 无连接、开销小、不保证可靠和有序，适合实时音视频、DNS、广播、多播等场景。
- TCP 可靠性来自确认应答、重传、序号、滑动窗口、拥塞控制等机制。

## 8. HTTP 请求报文和响应报文分别由哪些部分组成？

**答案要点：**

HTTP 请求报文通常包括：请求行、请求头、空行、请求体。

HTTP 响应报文通常包括：状态行、响应头、空行、响应体。

面试中可补充：

- 常见请求方法：GET、POST、PUT、DELETE、PATCH。
- 常见状态码：200、301、302、400、401、403、404、500、502、503。
- HTTP 是应用层协议，通常基于 TCP。

## 9. GET 和 POST 有什么区别？

**答案要点：**

- GET 常用于查询资源，参数常放在 URL 中；POST 常用于提交数据，请求体承载数据。
- GET 语义上应是安全、幂等的；POST 通常不是幂等的。
- GET 请求容易被缓存、收藏和记录日志；POST 更适合传输较大或结构化数据。
- 安全性不能只依赖 GET/POST，敏感数据应使用 HTTPS，并避免暴露在 URL 中。

## 10. 闭包是什么？有什么应用场景？

**答案要点：**

闭包是内部函数引用了外部函数作用域中的变量，并且外部函数已经返回后，这些变量仍然被内部函数持有。

常见应用：

- 装饰器。
- 函数工厂。
- 保存状态。
- 延迟计算。

## 11. 装饰器的本质是什么？

**答案要点：**

装饰器本质是一个可调用对象，用于接收函数或类并返回新的函数或类，从而在不修改原始代码的情况下增强行为。

典型应用：

- 日志记录。
- 权限校验。
- 性能统计。
- 缓存。
- 参数校验。

面试中建议补充 `functools.wraps`，它可以保留原函数名称、文档字符串等元信息。

## 12. 什么是上下文管理器？`with` 语句的作用是什么？

**答案要点：**

上下文管理器用于管理资源的进入和退出逻辑，核心协议是 `__enter__` 和 `__exit__`。

常见场景：

- 文件自动关闭。
- 数据库连接释放。
- 锁自动释放。
- 临时修改环境后自动恢复。

`with` 的优势是即使代码块中发生异常，也能执行资源清理逻辑。

## 13. 生成器和普通函数有什么区别？

**答案要点：**

- 普通函数调用后一次性执行并返回结果。
- 生成器函数使用 `yield`，调用后返回生成器对象，迭代时才逐步执行。
- 生成器适合处理大数据流、惰性计算和无限序列。
- 生成器可以节省内存，但只能按迭代顺序消费。

## 14. 深拷贝和浅拷贝有什么区别？

**答案要点：**

- 浅拷贝只复制最外层对象，内部可变对象仍然共享引用。
- 深拷贝会递归复制内部对象，生成相互独立的数据结构。
- Python 中常用 `copy.copy()` 和 `copy.deepcopy()`。
- 面试中可结合嵌套列表说明：外层列表不同，内层列表是否共享是关键。

## 15. Linux 中软链接和硬链接有什么区别？

**答案要点：**

- 硬链接指向同一个 inode，不能跨文件系统，通常不能链接目录。
- 软链接保存目标路径，类似快捷方式，可以跨文件系统，也可以链接目录。
- 删除原文件后，硬链接仍可访问文件内容；软链接会失效。
- 常用命令：`ln source target` 创建硬链接，`ln -s source target` 创建软链接。

## 16. Linux 文件权限如何理解？

**答案要点：**

Linux 权限主要分为用户、用户组、其他人三类，每类包含读、写、执行权限。

- `r`：读权限。
- `w`：写权限。
- `x`：执行权限。
- 文件权限常用八进制表示，例如 `755` 表示所有者可读写执行，组和其他人可读可执行。
- 对目录来说，`x` 表示能进入目录，`r` 表示能列出目录内容，`w` 表示能修改目录项。

## 17. 管道和重定向有什么区别？

**答案要点：**

- 管道 `|` 用于把前一个命令的标准输出作为后一个命令的标准输入。
- 输出重定向 `>` 覆盖写入文件，`>>` 追加写入文件。
- 输入重定向 `<` 从文件读取输入。
- 错误重定向可以使用 `2>`，标准输出和错误一起重定向可以使用 `&>` 或 `2>&1`。

## 18. Linux 排查 CPU、内存、磁盘问题常用哪些命令？

**答案要点：**

- CPU：`top`、`htop`、`ps`。
- 内存：`free`、`vmstat`、`top`。
- 磁盘空间：`df`。
- 目录占用：`du`。
- 磁盘 I/O：`iostat`。
- 网络连接：`ss`、`netstat`、`lsof`。
- 日志：`tail`、`journalctl`、`dmesg`。

## 19. Shell 脚本中 `$?`、`$0`、`$1`、`$#` 分别表示什么？

**答案要点：**

- `$?`：上一条命令的退出状态码。
- `$0`：当前脚本名称。
- `$1`、`$2`：第 1、2 个命令行参数。
- `$#`：参数个数。
- `$@`：所有参数，常用于遍历参数列表。

## 20. vim 常见模式有哪些？

**答案要点：**

- 普通模式：移动光标、复制、粘贴、删除。
- 插入模式：编辑文本。
- 命令模式：保存、退出、查找、替换。
- 常用命令：`i` 进入插入模式，`Esc` 返回普通模式，`:w` 保存，`:q` 退出，`:wq` 保存退出，`:q!` 强制退出。
