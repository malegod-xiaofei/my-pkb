# 02 数据结构与算法面试题

> 参考笔记：`AI黑马/01 基础知识/02 数据结构与算法（Python）`、`AI黑马/传智项目文档/10 基础算法面试题`。
>
> 权威参考：CLRS、Algorithms 4th、Python `collections`、`heapq`、`bisect` 官方文档。

## 1. 如何分析算法的时间复杂度？

**答案要点：**

时间复杂度描述输入规模增长时，算法执行时间的增长趋势。分析时通常保留最高阶项，忽略常数和低阶项。

常见复杂度从低到高：

- `O(1)`：常数时间。
- `O(log n)`：对数时间，常见于二分查找。
- `O(n)`：线性时间。
- `O(n log n)`：常见于高效排序。
- `O(n^2)`：双重循环。
- `O(2^n)`、`O(n!)`：指数或阶乘级，常见于暴力搜索。

## 2. 时间复杂度和空间复杂度有什么区别？

**答案要点：**

- 时间复杂度衡量算法运行时间随输入规模增长的变化趋势。
- 空间复杂度衡量算法额外占用内存随输入规模增长的变化趋势。
- 面试中需要关注“额外空间”，例如原地排序通常额外空间较小。

## 3. 数组和链表有什么区别？

**答案要点：**

- 数组内存连续，支持下标随机访问，查询快，插入删除可能需要移动元素。
- 链表内存不一定连续，不能高效随机访问，但在已知节点位置时插入删除更方便。
- 数组适合查询多、数据规模相对固定的场景。
- 链表适合频繁插入删除、数据规模动态变化的场景。

## 4. 栈和队列分别适合解决什么问题？

**答案要点：**

- 栈是后进先出，适合括号匹配、函数调用栈、表达式求值、DFS、撤销操作。
- 队列是先进先出，适合 BFS、任务调度、消息缓冲。
- 双端队列支持两端插入和删除，Python 中常用 `collections.deque`。

## 5. 哈希表的核心思想是什么？

**答案要点：**

哈希表通过哈希函数把 key 映射到数组下标，从而实现平均 `O(1)` 的查找、插入和删除。

需要关注：

- 哈希冲突处理：链地址法、开放寻址法。
- 装载因子：元素数量与桶数量的比例。
- 哈希函数质量：尽量均匀分布，减少冲突。
- Python 中 `dict` 和 `set` 都基于哈希表思想。

## 6. 二叉树有哪些常见遍历方式？

**答案要点：**

- 前序遍历：根、左、右。
- 中序遍历：左、根、右。
- 后序遍历：左、右、根。
- 层序遍历：按层从左到右，通常使用队列。

二叉搜索树的中序遍历结果是有序序列。

## 7. 快速排序的思想是什么？复杂度如何？

**答案要点：**

快速排序通过选择基准值，将数组划分为小于基准和大于基准的两部分，然后递归排序。

- 平均时间复杂度：`O(n log n)`。
- 最坏时间复杂度：`O(n^2)`，例如每次划分极不均衡。
- 空间复杂度：递归栈平均 `O(log n)`，最坏 `O(n)`。
- 优化方式：随机选基准、三数取中、三路快排。

## 8. 冒泡排序、选择排序、插入排序有什么区别？

**答案要点：**

- 冒泡排序：相邻元素比较并交换，每轮把最大或最小元素移动到末尾。
- 选择排序：每轮选择最小元素放到当前位置。
- 插入排序：把当前元素插入到前面已经有序的序列中。
- 三者平均时间复杂度通常都是 `O(n^2)`。
- 插入排序在近乎有序的数据上表现较好。

## 9. 排序算法的稳定性是什么意思？

**答案要点：**

如果两个元素的关键字相等，排序后它们的相对顺序保持不变，则该排序算法是稳定的。

- 稳定排序：冒泡排序、插入排序、归并排序。
- 不稳定排序：选择排序、快速排序、堆排序。
- 当数据存在多个排序关键字时，稳定性很重要。

## 10. 二分查找的前提是什么？容易错在哪里？

**答案要点：**

二分查找要求数据有序，并且可以随机访问。

常见易错点：

- 边界条件：左闭右闭还是左闭右开。
- `mid` 计算防溢出：`left + (right - left) // 2`。
- 循环条件：`left <= right` 或 `left < right`。
- 查找第一个或最后一个满足条件的位置时，需要继续收缩边界。

## 11. BFS 和 DFS 有什么区别？

**答案要点：**

- BFS 广度优先，通常使用队列，适合最短路径、层序遍历。
- DFS 深度优先，通常使用递归或栈，适合路径搜索、连通性、回溯。
- BFS 空间消耗可能较大，因为需要保存一层节点。
- DFS 递归实现简单，但需要注意递归深度。

## 12. 什么是递归？递归需要满足哪些条件？

**答案要点：**

递归是函数直接或间接调用自身来解决问题。

需要满足：

- 问题可以拆分为规模更小的同类子问题。
- 存在明确的终止条件。
- 每次递归都应向终止条件靠近。

面试中要注意递归栈空间和重复计算问题。

## 13. 动态规划适合解决什么问题？

**答案要点：**

动态规划适合具有最优子结构和重叠子问题的问题。

常见步骤：

1. 定义状态。
2. 写出状态转移方程。
3. 确定初始化条件。
4. 确定遍历顺序。
5. 返回最终结果。

典型题：背包问题、最长递增子序列、最长公共子序列、爬楼梯。

## 14. 贪心算法和动态规划有什么区别？

**答案要点：**

- 贪心算法每一步选择当前最优，希望最终得到全局最优。
- 动态规划会系统保存子问题结果，通常能处理更复杂的全局最优问题。
- 贪心需要证明局部最优能推出全局最优。
- 动态规划通常复杂度更高，但适用范围更广。

## 15. 如何反转单链表？

**答案要点：**

常用迭代法维护三个指针：

- `prev`：前一个节点。
- `cur`：当前节点。
- `next`：临时保存当前节点的下一个节点。

每次把 `cur.next` 指向 `prev`，然后三个指针向前移动，最后返回 `prev`。

时间复杂度 `O(n)`，空间复杂度 `O(1)`。

## 16. 如何用两个栈实现队列？

**答案要点：**

- 一个栈用于入队，另一个栈用于出队。
- 入队时直接压入输入栈。
- 出队时如果输出栈为空，则把输入栈元素全部弹出并压入输出栈，再从输出栈弹出。
- 每个元素最多进出两个栈，均摊时间复杂度为 `O(1)`。

## 17. TopK 问题如何解决？

**答案要点：**

常见方案：

- 排序：时间复杂度 `O(n log n)`，实现简单。
- 小顶堆：维护大小为 K 的堆，时间复杂度 `O(n log k)`，适合海量数据。
- 快速选择：平均时间复杂度 `O(n)`，但最坏 `O(n^2)`。

如果数据无法一次放入内存，可以结合分片、堆和外部排序。

## 18. Python 中 `list`、`deque`、`heapq` 分别适合什么场景？

**答案要点：**

- `list`：动态数组，适合随机访问和尾部追加。
- `deque`：双端队列，适合两端高效插入删除。
- `heapq`：堆结构，适合优先队列、TopK、合并有序序列。

面试中可补充：`list.pop(0)` 是 `O(n)`，队列场景更推荐 `deque.popleft()`。
