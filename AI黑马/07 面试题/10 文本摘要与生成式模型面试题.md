# 10 文本摘要与生成式模型面试题

> 参考笔记：`AI黑马/06 文本摘要(HTML)`。
>
> 权威参考：TextRank 论文、Pointer-Generator Networks 论文、ROUGE 论文、Hugging Face Transformers 文档。

## 1. 文本摘要任务是什么？

**答案要点：**

文本摘要是从长文本中生成简短摘要，保留主要信息并减少冗余。

常见类型：

- 抽取式摘要：从原文中选择重要句子或片段。
- 生成式摘要：生成新的自然语言表达，可能不完全复制原文。

## 2. 抽取式摘要和生成式摘要有什么区别？

**答案要点：**

- 抽取式摘要直接选择原文内容，事实一致性通常更好，但表达灵活性较弱。
- 生成式摘要可以重组语言，表达更自然，但可能出现幻觉或事实错误。
- TextRank 是典型抽取式方法。
- Seq2Seq、PGN、BART、T5 是典型生成式方法。

## 3. TextRank 的基本思想是什么？

**答案要点：**

TextRank 借鉴 PageRank 思想，把句子看作图中的节点，句子相似度作为边权，通过迭代计算句子重要性。

流程：

1. 文本切分为句子。
2. 计算句子之间相似度。
3. 构建句子图。
4. 迭代计算句子权重。
5. 选择权重最高的句子组成摘要。

## 4. TextRank 有什么优缺点？

**答案要点：**

优点：

- 无监督，不需要标注数据。
- 实现简单，可解释性较好。
- 适合作为摘要 baseline。

缺点：

- 依赖句子相似度质量。
- 不会生成新表达。
- 可能选择冗余句子。
- 对深层语义理解有限。

## 5. Seq2Seq 用于摘要有什么问题？

**答案要点：**

常见问题：

- 长文本信息压缩困难。
- 容易重复生成。
- OOV 词处理较弱。
- 事实一致性难保证。
- 长文本训练和推理成本较高。

Attention、Copy 机制、Coverage 机制可以缓解部分问题。

## 6. Attention 在摘要中的作用是什么？

**答案要点：**

Attention 让解码器在生成每个词时关注原文不同位置的信息。

作用：

- 缓解固定向量瓶颈。
- 提升长文本信息利用能力。
- 可视化注意力权重，提供一定解释性。
- 是 PGN 和 Transformer 摘要模型的重要基础。

## 7. Pointer-Generator Network 的核心思想是什么？

**答案要点：**

PGN 结合生成模式和复制模式。

- 生成模式：从词表中生成词。
- 指针模式：从原文中复制词。
- 通过生成概率控制当前步骤更偏向生成还是复制。

PGN 能更好处理 OOV、实体名、数字等需要从原文复制的信息。

## 8. Coverage 机制解决什么问题？

**答案要点：**

Coverage 机制记录历史注意力分布，减少模型反复关注同一原文片段，从而缓解重复生成问题。

- 对摘要重复句、重复短语有帮助。
- 通常配合 coverage loss 使用。
- 可以提升摘要多样性和覆盖度。

## 9. Beam Search 的作用是什么？

**答案要点：**

Beam Search 在解码时保留多个候选序列，而不是每一步只选择概率最高的词。

- beam size 越大，搜索空间越大，但计算成本越高。
- 太大的 beam size 可能生成保守或重复的摘要。
- 常用于翻译、摘要等序列生成任务。

## 10. Beam Search 和 Greedy Search 有什么区别？

**答案要点：**

- Greedy Search 每一步选择当前概率最高的 token，速度快但容易陷入局部最优。
- Beam Search 保留多个候选路径，整体质量通常更好。
- Beam Search 计算成本更高，需要设置 beam size。

## 11. ROUGE 指标是什么？

**答案要点：**

ROUGE 用于评估自动摘要和参考摘要之间的重合程度。

常见类型：

- ROUGE-1：一元词重合。
- ROUGE-2：二元词重合。
- ROUGE-L：最长公共子序列。

ROUGE 主要衡量词面重合，不能完全代表摘要事实性和可读性。

## 12. BLEU 和 ROUGE 有什么区别？

**答案要点：**

- BLEU 常用于机器翻译，更关注生成文本 n-gram 精确率。
- ROUGE 常用于文本摘要，更关注参考摘要内容被覆盖的程度。
- 二者都是自动指标，都无法完全替代人工评估。

## 13. 生成式摘要为什么会出现事实错误？

**答案要点：**

原因可能包括：

- 模型根据语言模式生成，而不是严格复制事实。
- 原文长、关键信息分散。
- 训练数据中存在噪声。
- 解码策略导致模型偏向流畅但不准确的表达。
- 缺少事实约束或外部校验。

## 14. 如何缓解生成式摘要的事实错误？

**答案要点：**

- 引入 Copy 或 Pointer 机制。
- 使用检索或证据约束。
- 增加事实一致性评估和 rerank。
- 使用实体一致性检查。
- 加强训练数据清洗。
- 高风险场景加入人工审核。

## 15. 长文本摘要有什么难点？

**答案要点：**

- 输入长度超过模型上下文窗口。
- 关键信息分散在多个段落。
- 长序列 Attention 成本高。
- 摘要需要兼顾覆盖度和简洁性。

解决思路：

- 分块摘要再汇总。
- 层次化摘要。
- 长文本 Transformer。
- 检索增强摘要。

## 16. 文本摘要项目如何做 baseline？

**答案要点：**

可选 baseline：

- Lead-3：取前三句。
- TextRank：抽取式图排序。
- 普通 Seq2Seq + Attention。
- 预训练模型微调。

Baseline 的作用是提供可比较的起点，后续优化需要证明相对提升。

## 17. 文本摘要模型上线需要关注什么？

**答案要点：**

- 推理延迟。
- 输入长度限制。
- 摘要长度控制。
- 并发能力。
- 事实一致性。
- 敏感内容过滤。
- 失败兜底策略。
- 监控摘要质量和用户反馈。

## 18. 如何介绍文本摘要项目？

**答案要点：**

可以按以下结构回答：

1. 项目目标和应用场景。
2. 数据来源、样本格式和摘要类型。
3. Baseline：TextRank 或 Seq2Seq。
4. 模型方案：Attention、PGN、Coverage 或预训练模型。
5. 评估指标：ROUGE + 人工评估。
6. 难点：重复生成、OOV、事实一致性、长文本。
7. 优化：Coverage、Beam Search、数据增强、部署加速。
