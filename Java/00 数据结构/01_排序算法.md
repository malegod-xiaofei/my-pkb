- 快速排序
  - 时间复杂度O(nlogn)
  - 最坏情况下的时间复杂度O(n^2)--有序的数列用快速排序
- 快速排序解决最坏情况的方法
  - 方法一、打乱（也就是数组的shuffle阶段进行打乱，）
  - 方法二、选择分区第一个、中间一个、最后一个元素的中直作为枢纽。可以大大增加快速排序性能。
- 双向链表转换为二叉树？
  - 算法思路：
  - 首先遍历链表，统计结点个数
  - 首先我们通过下标找出中间的那个数作为根节点，假设下标为x
  - 那么下标0 ~ x-1的数肯定比x小，x+1 ~ n-1的数比x大，将其分成三段
  - 新建根节点root的值为下标x的数值，然后root的左孩子的值就是等于下标0 ~ x-1的中间的下标的值，而右孩子就是x+1 ~ n 中间的下标的值，去到这一步大家都知道其实就是重复2跟着3，不停的递归下去，只要设置好递归的退出就能轻易的转换好这个BST。
