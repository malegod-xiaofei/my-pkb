# 3 3.多态的几种实现形式

- 概念：同一操作作用于不同的对象，可以有不同的解释，产生不同的执行结果，这就是多态
- 特征特点：为了代码的重用，项目耦合度很高的情况下实现解耦，程序的可替换性，可扩充性
  - 首先：多态存在的三个必要条件
    - 要有继承关系
    - 要有重写
    - 父类引用指向子类对象
  - 普通类：父类引用指向子类实体
  - 抽象类：抽象类变量指向子类实体
  - 接口：接口变量指向子类实体
- 应用场景：比如我们在实际开发过程中，常常遇到一个功能有多重实现方式比如支付方式，有分京东、微信、支付宝、银联等等，不同支付方式的大概流程相似，实现细节有所区别。这个时候用java的多态，定义一个公共接口，接口定义支付流程的各个方法，具体支付方式实现具体的接口方法
- 代码实现

# 4 4.synchronized和lock的区别？

- lock不是java语言内置的，synchronized是Java语言的关键字，因此是内置特性。Lock是一个类，通过这个类可以实现同步访问
- 采用synchronized不需要用户去手动释放锁，当synchronized方法或者synchronized代码块执行完之后，系统会自动让线程释放对锁的占用；而Lock则必须要用户去手动释放锁，如果没有主动释放锁，那么会出现死锁的现象。

# 6 6.生产者、消费者模型的作用？

- 通过平衡生产者和消费者的消费能力来提高整个系统的运行效率
- 如何在两个线程之间共享数据
  - 方法一、将数据抽象成一个类，并将对这个数据的操作作为这个类的方法。
  - 方法二、将Runnable对象作为一个类的内部类，共享数据作为这个类的成员变量，每个线程对共享数据的操作方法也封装在外部类，以便实现对数据的各个操作的同步和互斥，作为内部类的各个Runnable对象调用外部类的这些方法。

# 7 7.重载和重写的区别和联系

- 重载发成在类内，要求方法名相同，返回值相同，重载的两种形式，一个是参数个数不同，一种是个数相同，类型不同
- 重写发成在类间，要求有继承关系，子类比父类有更宽泛的异常，不能有更低的访问权限，方法名，参数列表，返回值必须一致，重写特指成员方法
