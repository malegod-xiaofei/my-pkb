一、任务背景

- 在团队协同开发项目中，每一次提交合并到 master 的 Pull Request，有且仅对应一条 commit 和 log。但是提交合并的请求不一定一次过，有些 Pull Request就对应着多条 commit 和多条 log 这在后期的代码 review 不太友好，所以需要将单 Pull Request中的 commit 合并成一个
  - 在Git中，如果你想将最近的N次提交合并为一次提交，你可以使用git rebase命令的 interactive 模式（即git rebase -i）。下面是具体的操作步骤：
    - 查看提交历史：
      - 首先，确认你的工作目录是干净的，没有未保存的更改。然后，使用 git log 查看提交历史，记住你想要合并的那两次提交的哈希值或前几位字符。
      - 启动交互式重置：
      - 接下来，运行以下命令来启动交互式重置模式，这里HEAD~2表示你要编辑的最近两次提交：

```bash
git rebase -i HEAD~2
```

    - 编辑提交列表：
      - 这时，你的默认文本编辑器会打开一个包含最近两次提交信息的列表。这个列表每一行对应一个提交，看起来像这样：

```
pick <commit_hash_1> Commit message 1
pick <commit_hash_2> Commit message 2
```

      - 要将这两次提交合并为一次，你需要将"pick"改为"squash"或"s"（代表压缩）在第二个提交前，结果如下：

```
pick <commit_hash_1> Commit message 1
squash <commit_hash_2> Commit message 2
```

      - 这意味着第二个提交将会被压缩到第一个提交中。
      - 保存并退出编辑器：
        - 保存文件并关闭编辑器。ESC->Shift->:，输入wq，最后按 Enter 键退出。
    - 编辑合并后的提交信息：
      - 之后，Git会再次打开一个编辑器，让你有机会编辑这次合并后的提交信息。默认情况下，它会显示两次提交信息的组合。你可以根据需要编辑这个消息，然后保存并关闭编辑器。
    - 完成重置：
    - 完成上述步骤后，Git会应用这个新的、合并过的提交，并且原来的两次提交会被替换为一次。你可以用 git log 来检查结果。
    - 最后将合并好的log推送到远程分支

```bash
git push origin dev -f
#如果上面是master可能会报错，因为默认master是受保护的，需要去gitlab里面先去取消受保护的分支
```

常用的 git 命令:

这里简单列出, 每个命令的具体行为和参数, 要看git help , 和实际操作.

- git status: 查看本地状态
- git log: 查看提交历史
  - git log --color --graph --decorate -M --pretty=oneline --abbrev-commit -M 以图的形式显示提交历史. 增加 --all参数可以看到所有分支的历史. 类似的长命令可以配置快捷方式, 参见 gitconfig
- git branch: 列出分支或创建分支等.
- git checkout: 切换分支, 撤销修改等.
- git checkout -b: 新建分支并切换到新建立的分支.
- git rebase: 重建提交历史
  - git rebase base_branch tip_branch: 把 tip_branch 以下所有没有包含在base_branch中的提交点, 放到base_branch 上面.
  - git rebase --onto base_branch from_branch tip_branch. rebase 的完整参数形式. 指定from_branch到tip_branch的所有提交点, 放到 base_branch上面.
- git rebase -i: 重新调整提交点顺序.
- git reset: 撤销本地修改
- git reset --hard: 撤销本地修改(包括 staged 的修改). 也可以用于强制切换到某个分支.
- git fetch: 从远端(譬如 origin) 拉取修改, 但不修改本地内容.
- git merge --ff-only:merge 直接父子关系的分支.
- git push: 推送提交历史到远端.
  - git push origin branch: 指定推送的本地分支
  - git push origin branch:remote_branch: 指定本地分支branch推送到远端分支remote_branch
  - git pull
