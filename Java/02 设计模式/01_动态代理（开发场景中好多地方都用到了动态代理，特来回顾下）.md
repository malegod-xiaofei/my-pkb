1、什么是动态代理？

- 概念
  - 动态代理（基于反射机制）是设计模式当中代理模式的一种；要搞清楚什么是动态代理得先知道静态代理（静态代理事实上就是装饰者模式，区别是装饰者模式一般都是传入一个被装饰类在构造器中，而静态代理则是自己创建一个类作为成员变量），动态代理就是程序运行时创建的代理方式被称为动态代理
  - 在程序的执行过程中，使用jdk的反射机制，创建代理类对象，并动态的指定要代理目标类，换句话说：动态代理是一种创建Java对象的能力，让你不用创建TaoBao类，就能创建代理对象
- 特征特点
- 应用场景
- 代码实现
  - 1）JDK动态代理（掌握）：使用Java反射包中的类和接口实现动态代理的功能。反射包java.lang.reflect，里面有三个类：InvocationHandler,Method,Proxy,
  - 2)cglib动态代理（理解）：cglib是第三方的工具库，创建代理对象。
    - cglib的原理是继承，cglib通过继承目标类，创建它的子类，在子类中重写父类中同名的方法，实现功能的修改；
    - 因为cglib是继承，重写方法，所以要求目标类不能是final的，方法也不能是final的。cglib要求目标类比较宽松，只要能继承就可以了。cglib在很多的框架中使用，比如：mybatis，spring框架中都有使用
