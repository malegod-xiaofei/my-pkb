- git rebase 报错 Cannot rebase: You have unstaged changes.

```bash
# 意味着你的工作目录中有未暂存的更改，可以暂存更改但不提交，（如果你想保留更改以便在 rebase 后继续工作）
# 使用 git stash 来临时存储未提交的更改：
git stash

# 完成 rebase 操作后，你可以通过以下命令恢复暂存的更改：
git stash pop
```

- git的工作流程

```bash
# 1.自报家门
git config -global user.name…
# 2.初始化仓库目录
git init
# 3.加入缓存区
git add
# 4.加入本地仓库
git commit -m ""
# 5.推送到远程仓库
git remote add name git_url
git push origin master
# 6.拉取代码并合并
git fetch
git merge
git pull=fetch + merge
# 7.代码的对比或差异分析
git diff
# 8.克隆项目
git clone git_url
```

- Git ignore机制
不是所有代码及相关的文件都需要版本化管理，比如日志、编译完的jar包、残留历史文件等等，但是某些IDE会提示有未进入版本管理的文件，使得状态不准确

解决该问题，只需要将确定不需要版本管理的文件加入.gitignore配置文件即可
