# 1 1.SpringBoot常用注解

- @SpringBootApplication：声明为springboot的启动类
- @ComponentScan：管理包下的所有Component注解，包含Component的子注解，比如，service，Controller
- @Component：放在类上，把普通类实例化放到spring容器中。
- @Bean：放在方法上，用@Bean标注方法等价于XML中配置bean，这个方法一般返回一个实体对象，告诉Spring这里产生一个对象，然后这个对象会交给spring管理。
- @Configuration：标注当前类是配置类，并会将当前类内声明的一个或多个以@Bean注解标记的方法的实例纳入到spring容器中，并且实例名就是方法名。
- @ConfigurationProperties：将配置文件中的参数映射成一个对象，通过prefix来设定前缀，然后将后面的和对象的属性名一致就能实现注入
- @Autowried：默认是按照类型进行装配注入，如果允许null值，可以把设置它 required为false，等于告诉spring，在找不到匹配Bean时也不报错。
- @RequestMapping：浏览器请求的url地址
- @ResponseBody：将返回的Java对象转为json格式
- @RequestParam：主要用于将请求参数区域的数据映射到控制层方法的参数上

# 2 2.什么叫SpringBoot？

- 概念：开源的轻量级框架，约定俗成，也就是约定大于配置，一切自动完成，可以大大简化你的开发模式，springboot简化了我们的配置依赖，可以自动启动我们想要的所有功能，并且只需要点击一下就能启动应用程序
- 特征特点：springboot并不是一个全新的框架，而是将已有的spring组件整合起来，约定俗成，简单，快速，便捷。可以在几秒内构建一个项目，写好gav，一、groupid：我是谁，项目的唯一坐标，若定位到git上通过g和a找到项目。域名比如com.tianliang.zyf二、artifactId：项目名，模块名；三、version：是springboot的版本号
- 应用场景：

  - 2.J2EE应用，3.微服务架构
