# HBase 高频面试题

## 整理依据

基于本目录 HBase 知识点与常用操作笔记整理，重点覆盖 HBase 数据模型、架构组件、Shell 与 Java API、读写流程、MemStore/WAL/HFile、Compaction、Region Split、RowKey 设计与生产优化。

## 一、基础概念与数据模型

### 1. HBase 是什么？适合什么场景？

HBase 是基于 HDFS 的分布式、可扩展、面向列族的 NoSQL 数据库，适合海量明细数据存储和按 RowKey 的低延迟随机读写场景，例如日志明细、用户画像、订单轨迹、监控数据等。

### 2. HBase 和传统关系型数据库有什么区别？

HBase 不强调复杂事务和多表 Join，数据按 RowKey 字典序分布存储，面向列族，适合海量稀疏数据和水平扩展。关系型数据库更适合强事务、复杂 SQL 和多表关系查询。

### 3. HBase 的数据模型包括哪些核心概念？

核心概念包括 Namespace、Table、RowKey、Column Family、Column Qualifier、Timestamp 和 Cell。一个 Cell 由 RowKey、列族、列限定符、时间戳和值共同定位。

### 4. 为什么说 HBase 是稀疏、多维、排序 Map？

HBase 中每行可以拥有不同列，空列不占存储空间，因此是稀疏的；数据通过 RowKey、列族、列、时间戳多维定位；同一表内数据按 RowKey 字典序有序存储。

### 5. Column Family 和 Column Qualifier 有什么区别？

Column Family 是建表时定义的列族，是物理存储和权限、压缩等配置的基本单位；Column Qualifier 是列族下的具体列名，可动态增加。列族不宜过多，列可以很多。

### 6. HBase 中 Timestamp 有什么作用？

Timestamp 用于区分同一 Cell 的多个版本。HBase 可以保留多个版本数据，查询时可指定版本数或时间范围；默认通常返回最新版本。

## 二、架构组件

### 7. HBase 由哪些核心组件组成？

核心组件包括 HMaster、RegionServer、ZooKeeper 和 HDFS。HMaster 负责表和 Region 管理；RegionServer 负责读写请求和 Region 服务；ZooKeeper 负责协调；HDFS 负责底层持久化存储。

### 8. HMaster 的主要职责是什么？

HMaster 负责表的创建、删除、变更，Region 分配，RegionServer 故障恢复协调，负载均衡等。实际读写请求通常不经过 HMaster，而是由客户端直接访问 RegionServer。

### 9. RegionServer 的主要职责是什么？

RegionServer 负责管理多个 Region，处理客户端读写请求，维护 MemStore、WAL、BlockCache，执行 Flush、Compaction 和 Split 等操作。

### 10. ZooKeeper 在 HBase 中有什么作用？

ZooKeeper 用于维护集群协调信息，例如 Master 选举、RegionServer 状态、元数据入口等。客户端通常通过 ZooKeeper 获取 `hbase:meta` 位置，再定位业务数据所在 Region。

### 11. `hbase:meta` 表保存什么信息？

`hbase:meta` 保存用户表 Region 的位置信息，包括表名、Region 范围、RegionServer 地址等。客户端定位某个 RowKey 时会先查 meta，再访问对应 RegionServer。

## 三、常用操作与 API

### 12. HBase Shell 中如何创建命名空间和表？

常用命令包括 `create_namespace 'bigdata'` 创建命名空间，`create 'bigdata:person', {NAME => 'info', VERSIONS => 5}, {NAME => 'msg'}` 创建带列族和版本配置的表。

### 13. HBase Shell 中 put、get、scan、delete 有什么区别？

`put` 写入指定 RowKey、列族、列和值；`get` 按 RowKey 查询单行；`scan` 按范围扫描多行；`delete` 删除某个单元格或版本；`deleteall` 可删除整行或指定列的全部版本。

### 14. 删除 HBase 表为什么通常要先 disable？

HBase 删除表前需要先禁用表，避免表仍在提供读写服务时被删除。常见流程是 `disable 'table'` 后再 `drop 'table'`。

### 15. Java API 操作 HBase 的核心对象有哪些？

常见对象包括 `Connection`、`Admin`、`Table`、`Put`、`Get`、`Scan`、`Delete`、`Result`、`ResultScanner`。`Admin` 用于 DDL，`Table` 用于 DML。

### 16. Scan 查询为什么要尽量指定 StartRow 和 StopRow？

HBase 按 RowKey 有序存储，指定 StartRow 和 StopRow 可以只扫描目标范围，避免全表扫描。RowKey 设计合理时，范围扫描效率很高。

## 四、读写流程与存储结构

### 17. HBase 写数据的基本流程是什么？

客户端定位 RegionServer 后发送写请求，RegionServer 先写 WAL，再写 MemStore。WAL 保证故障恢复，MemStore 达到阈值后 Flush 成 HFile 持久化到 HDFS。

### 18. WAL 的作用是什么？

WAL 是预写日志，写入数据先记录 WAL，再写 MemStore。RegionServer 异常宕机时，可以通过 WAL 回放恢复尚未 Flush 到 HFile 的数据。

### 19. MemStore 的作用是什么？

MemStore 是内存写缓存，每个 Store 通常对应一个 MemStore。写入先进入 MemStore，达到阈值后 Flush 生成 HFile，从而把随机写转为顺序写，提高写入性能。

### 20. HFile 是什么？

HFile 是 HBase 在 HDFS 上的底层存储文件，按 KeyValue 有序存储。HBase 通过 HFile、索引、布隆过滤器、BlockCache 等机制提升查询性能。

### 21. BlockCache 的作用是什么？

BlockCache 用于缓存热点 HFile 数据块，提升读性能。读请求会优先查 MemStore，再查 BlockCache，最后再读取 HDFS 上的 HFile。

### 22. HBase 读数据的基本流程是什么？

客户端定位 RegionServer 后发起读请求，RegionServer 先查 MemStore，再查 BlockCache，最后查 HFile，并将多个来源的数据按版本、时间戳、删除标记等规则合并后返回。

### 23. Flush 会带来什么影响？

Flush 会把 MemStore 数据写成新的 HFile，释放内存并保证数据持久化。但频繁 Flush 会产生大量小 HFile，增加后续 Compaction 压力。

## 五、Compaction、Split 与 RowKey 优化

### 24. Minor Compaction 和 Major Compaction 有什么区别？

Minor Compaction 会合并部分小 HFile，减少文件数量；Major Compaction 会合并 Store 下所有 HFile，并清理删除标记和过期版本，资源消耗更大，生产中要避免在业务高峰频繁触发。

### 25. Region Split 是什么？

Region Split 是当 Region 数据量增长到阈值后，将一个 Region 拆分为两个 Region，以便分散到不同 RegionServer 上，提高并发和容量扩展能力。

### 26. 什么是预分区？为什么需要预分区？

预分区是在建表时提前划分多个 Region，避免新表初期只有一个 Region 导致所有写入集中到单个 RegionServer。大规模写入前通常结合 RowKey 规则设计预分区。

### 27. RowKey 设计为什么很重要？

RowKey 决定数据排序、Region 分布和查询方式。好的 RowKey 能兼顾写入均衡和查询效率；差的 RowKey 可能造成热点、扫描范围过大或无法利用范围查询。

### 28. RowKey 热点问题如何产生？如何解决？

如果 RowKey 单调递增，如时间戳直接作为前缀，大量新数据会持续写入最后一个 Region，形成热点。解决方式包括加盐、Hash 前缀、反转、业务分桶、预分区等。

### 29. RowKey 设计有哪些常见原则？

常见原则包括长度适中、能唯一定位业务记录、常用查询字段靠前、避免单调递增导致热点、兼顾范围查询和散列均衡、尽量避免过长 RowKey 增加存储开销。

### 30. HBase 查询优化通常从哪些方面入手？

优先优化 RowKey 和 Scan 范围，避免全表扫描；合理设置列族和版本数；使用过滤器减少返回数据；利用 BlockCache 缓存热点；控制 HFile 数量和 Compaction；高并发写入前做好预分区。
