# Spark 高频面试题

## 整理依据

基于本目录 Spark 入门、常用算子、Spark 内核、Spark 优化、内核及调优等笔记整理，重点覆盖 RDD、任务调度、Shuffle、内存管理、算子选择、数据倾斜、资源配置与生产故障排查。

## 一、基础概念

### 1. Spark 的核心组件有哪些？

Driver 负责执行用户程序的 main 方法、生成 Job、调度 Task、跟踪 Executor 状态并提供 UI；Executor 负责执行具体 Task、缓存 RDD 分区并向 Driver 汇报状态；Cluster Manager 负责资源管理，如 Standalone、YARN、K8S 等。

### 2. 什么是 RDD？它有哪些核心特性？

RDD 是弹性分布式数据集，是 Spark 最基础的数据抽象。它具备分区、不可变、可并行计算、可基于血缘恢复等特性。RDD 的转换操作是惰性执行的，只有遇到 Action 才会触发真正计算。

### 3. RDD 的血缘关系有什么作用？

血缘关系记录了 RDD 从数据源到当前结果的转换链路。它主要用于容错恢复：某个分区丢失时，可以根据 Lineage 重新计算；也用于 DAG 构建、依赖分析和 Stage 划分。

### 4. Spark 中 Transformation 和 Action 的区别是什么？

Transformation 生成新的 RDD，如 map、filter、flatMap、reduceByKey，具有惰性；Action 会触发作业执行并返回结果或写出数据，如 collect、count、reduce、saveAsTextFile。一个 Action 通常触发一个 Job。

### 5. 窄依赖和宽依赖有什么区别？

窄依赖是父 RDD 的一个分区最多被一个子 RDD 分区依赖，可以进行 pipeline 执行；宽依赖是父 RDD 的一个分区会被多个子 RDD 分区依赖，通常伴随 Shuffle。Spark 以宽依赖为边界划分 Stage。

### 6. Job、Stage、Task 的关系是什么？

一个 Spark 应用包含多个 Job；每个 Action 触发一个 Job；DAGScheduler 根据宽依赖把 Job 划分为多个 Stage；每个 Stage 按分区数生成多个 Task，一个分区通常对应一个 Task。

## 二、任务调度与运行机制

### 7. Spark 作业大致提交流程是什么？

任务提交后启动 Driver，Driver 向资源管理器注册并申请 Executor；Executor 启动后反向注册到 Driver；Driver 执行用户代码，遇到 Action 后生成 DAG，DAGScheduler 划分 Stage，TaskScheduler 将 Task 分发到 Executor 执行。

### 8. DAGScheduler 和 TaskScheduler 分别负责什么？

DAGScheduler 负责 Stage 级调度，根据 RDD 依赖关系划分 Stage，把 Stage 封装成 TaskSet；TaskScheduler 负责 Task 级调度，将 TaskSet 中的 Task 按本地化、资源、调度策略等分发到 Executor。

### 9. Spark 的本地化调度是什么？

Spark 遵循“移动计算优于移动数据”的原则，尽量让 Task 在数据所在节点执行。若目标节点资源不足，会等待一段时间，超过本地化等待阈值后降级到较低本地化级别执行。

### 10. FIFO 和 FAIR 调度策略有什么区别？

FIFO 是默认策略，按 Job 提交顺序调度；FAIR 会基于调度池、minShare、weight 等参数进行公平调度，适合多个任务共享集群资源的场景。

### 11. Spark 的失败重试和黑名单机制怎么理解？

Task 失败后，TaskSetManager 会记录失败次数，未超过阈值则重新调度；同时记录失败所在 Executor 和 Host，在一定时间内避免把同一 Task 再调度到该节点，降低重复失败概率。

### 12. YARN client 和 YARN cluster 模式有什么区别？

YARN client 模式下 Driver 运行在提交任务的客户端机器上，适合测试和查看日志；YARN cluster 模式下 Driver 运行在 YARN 集群内部，适合生产环境，避免客户端网络和稳定性影响任务。

## 三、算子与开发实践

### 13. map 和 mapPartitions 的区别是什么？

map 按元素逐条处理，内存压力小但函数调用次数多；mapPartitions 按分区批量处理，适合批量建立连接、批量写库等场景，性能更高，但单分区数据过大时容易 OOM。

### 14. reduceByKey 和 groupByKey 的区别是什么？

reduceByKey 会在 map 端进行预聚合，减少 Shuffle 落盘、网络传输和 reduce 端内存压力；groupByKey 只分组不聚合，会把所有 value 拉到 reduce 端。分组聚合场景优先使用 reduceByKey。

### 15. aggregateByKey、foldByKey、combineByKey 有什么区别？

aggregateByKey 支持分区内和分区间使用不同聚合逻辑；foldByKey 是分区内和分区间逻辑相同时的简化版；combineByKey 最通用，可在聚合时改变 value 的数据结构，如求平均值时转换为 `(sum,count)`。

### 16. coalesce 和 repartition 的区别是什么？

两者都能改变分区数。repartition 底层是 `coalesce(numPartitions, shuffle=true)`，一定发生 Shuffle；coalesce 默认不 Shuffle，常用于缩减分区，但分区数减少过多可能导致数据不均衡或 OOM。

### 17. 哪些操作通常会触发 Shuffle？

常见包括 repartition、coalesce 开启 shuffle、reduceByKey、groupByKey、sortByKey、distinct、join、cogroup 等。凡是需要跨分区重新分布数据的操作，通常都会触发 Shuffle。

### 18. collect 算子有什么风险？

collect 会把所有分区数据拉到 Driver 内存，数据量大时容易造成 Driver OOM。生产中应限制数据量，如使用 take、takeOrdered，或将结果写入外部存储。

## 四、Shuffle 与内存机制

### 19. Spark Shuffle 的核心流程是什么？

Shuffle 分为 Shuffle Write 和 Shuffle Read。上游 map 端按分区器将数据写成本地 Shuffle 文件，下游 reduce 端拉取属于自己的数据后进行聚合、排序或后续计算，过程中会有网络传输和磁盘落盘。

### 20. HashShuffle 和 SortShuffle 有什么区别？

HashShuffle 早期按 reduce 分区生成大量小文件，优化后通过复用 buffer 减少文件数；Spark 2.0 后删除 HashShuffle，主要使用 SortShuffle。SortShuffle 会先在内存中聚合或排序，达到阈值后溢写磁盘，最后合并临时文件并生成索引文件。

### 21. bypass SortShuffle 的触发条件是什么？

当 Shuffle reduce task 数量小于等于 `spark.shuffle.sort.bypassMergeThreshold`（默认 200），且不是聚合类 Shuffle 算子时，会触发 bypass 机制。它不进行排序，减少排序开销，但仍会产生临时文件再合并。

### 22. Spark 堆内内存和堆外内存有什么区别？

堆内内存由 JVM 管理，申请释放依赖 GC，Spark 只能逻辑估算对象占用；堆外内存直接向操作系统申请，数据通常以序列化形式存储，可精确计算和释放，能降低 GC 压力，但排查内存泄漏更困难。

### 23. Spark 统一内存管理是什么？

Spark 1.6 后引入统一内存管理，Storage 内存和 Execution 内存共享一块区域，可动态借用空闲空间。Execution 内存可要求 Storage 淘汰并归还空间，而 Storage 被 Execution 占用后通常无法强制让 Execution 归还。

### 24. RDD 持久化和 checkpoint 有什么区别？

persist/cache 主要用于性能优化，缓存数据但保留血缘，缓存丢失可按血缘重算；checkpoint 会把数据写入可靠存储并切断血缘，主要用于容错和截断过长血缘，但写入 HDFS 等存储会有额外开销。

## 五、性能调优与生产问题

### 25. Spark 常规性能调优有哪些原则？

合理配置资源；避免重复创建和重复计算 RDD；复用并持久化多次使用的 RDD；尽量提前 filter；减少 Shuffle；优先使用带 map 端预聚合的算子；广播大变量；使用 Kryo 序列化；优化数据结构和分区数。

### 26. Spark 资源参数如何理解？

常见参数包括 `--num-executors` 控制 Executor 数量，`--executor-memory` 控制 Executor 堆内内存，`--executor-cores` 控制每个 Executor CPU 核数，`--driver-memory` 控制 Driver 内存，`spark.yarn.executor.memoryOverhead` 控制 YARN 堆外开销。

### 27. 并行度如何设置更合理？

并行度过低会导致 CPU 空闲，过高会增加调度开销。经验上 Task 数量可设置为总 CPU core 的 2 到 3 倍。RDD 可通过 `spark.default.parallelism` 或算子参数控制；Spark SQL Shuffle 可通过 `spark.sql.shuffle.partitions` 控制。

### 28. 如何处理 Spark 数据倾斜？

先定位倾斜 Stage 和 Shuffle 算子，再结合场景处理：过滤少量异常 key；提高 reduce 并行度；两阶段聚合加随机前缀；小表广播转 map join；倾斜 key 单独处理；大表 join 可扩容与随机前缀打散；必要时在上游预聚合。

### 29. 如何判断是数据倾斜还是数据量过大？

数据倾斜表现为少数 Task 很慢或 OOM，大多数 Task 很快；数据量过大则通常所有 Task 都慢或内存压力都较大。可通过 Spark UI 查看各 Task 的输入数据量、运行时间、Shuffle read/write 和 GC 时间。

### 30. Spark OOM 常见原因有哪些？

常见原因包括单分区数据过大、数据倾斜、mapPartitions 一次处理过多数据、collect 拉取大量数据到 Driver、Shuffle read 缓冲过大、缓存过多、对象创建过多导致 GC 压力过大、Executor 堆外内存不足。

### 31. 遇到 shuffle file not found 或 fetch failed 怎么排查？

常见原因是上游 Executor 挂掉、GC 停顿导致响应超时、网络抖动或堆外内存不足。可增加 `spark.shuffle.io.maxRetries`、`spark.shuffle.io.retryWait`、`spark.core.connection.ack.wait.timeout`，并检查 Executor 内存、GC 和节点稳定性。

### 32. 写数据库时为什么推荐 foreachPartition？

foreach 每条数据创建连接代价过高；foreachPartition 可以每个分区创建一次连接并批量写入，显著降低连接创建和网络交互成本。但要控制分区大小，避免单分区数据过多导致内存或事务压力过大。
