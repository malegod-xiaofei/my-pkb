# Hive 高频面试题

## 整理依据

基于本目录内 Hive 基础、函数进阶、SQL 面试题与企业级调优笔记整理，重点覆盖 Hive 架构原理、常用函数、窗口函数、SQL 案例、执行计划、Join 优化、数据倾斜与生产问题。

## 一、基础架构与元数据

### 1. Hive 是什么？它和 Hadoop 的关系是什么？

Hive 是基于 Hadoop 的数据仓库工具，可以把结构化数据文件映射成表，并提供类 SQL 的 HQL 查询能力。Hive 本身不存储数据，数据通常存储在 HDFS，计算任务可转换为 MapReduce、Tez 或 Spark 任务并提交到 Yarn 执行。

### 2. Hive 的本质是什么？

Hive 本质上是 Hadoop 的客户端。它负责解析 HQL、生成执行计划，并将任务提交到底层计算引擎执行；它不负责真正的数据存储和分布式计算。

### 3. Hive 的核心架构包括哪些组件？

主要包括 Client、Driver、Metastore 和 Hadoop。Client 提供 CLI、beeline、JDBC 等访问方式；Driver 负责 SQL 解析、语义分析、优化、生成执行计划和执行；Metastore 保存库表、字段、分区、表类型、数据路径等元数据；Hadoop 提供 HDFS、Yarn 和计算引擎能力。

### 4. Metastore 保存哪些信息？为什么生产环境通常不用 Derby？

Metastore 保存数据库、表名、字段、分区字段、表类型、表数据路径、SerDe、权限等元数据信息。Derby 通常只适合本地单用户测试，生产环境多客户端并发访问时容易受限，因此一般使用 MySQL 等独立关系型数据库保存元数据。

### 5. Metastore 嵌入式模式和独立服务模式有什么区别？

嵌入式模式下 Hive 客户端直接连接元数据库，配置简单但不利于多客户端共享和服务治理。独立服务模式会启动 Metastore 服务，HiveServer2 或 Hive 客户端通过 Thrift 访问 Metastore，生产环境更常用。

### 6. HiveServer2 和 beeline 的作用是什么？

HiveServer2 提供远程 JDBC/ODBC 服务，支持多客户端并发连接、认证和权限控制。beeline 是连接 HiveServer2 的命令行客户端，生产中比直接使用 hive CLI 更常见。

### 7. Hive 参数配置优先级是什么？

常见优先级为：配置文件参数 < 启动命令行参数 < HQL 中 `set` 设置。临时会话参数优先级更高，但通常只对当前会话有效。

## 二、表设计与常用操作

### 8. Hive 内部表和外部表有什么区别？

内部表由 Hive 管理数据生命周期，删除表时元数据和数据通常都会被删除。外部表只由 Hive 管理元数据，删除表时一般只删除元数据，不删除 HDFS 上的原始数据。生产中对原始日志、共享数据通常使用外部表。

### 9. 分区表的作用是什么？

分区表把数据按日期、地区等字段划分到不同目录，查询时如果带分区条件可以减少扫描范围，提高查询效率。常见分区字段包括 `dt`、`hour`、`region` 等。

### 10. 分桶表适合什么场景？

分桶表按字段 Hash 后写入固定数量文件，适合抽样、Bucket Map Join、SMB Join 等场景。它能让相同分桶键的数据落到对应桶中，减少 Join 或抽样时的扫描和 Shuffle 成本。

### 11. Hive 表和 HDFS 文件之间是什么关系？

Hive 表本质上是 HDFS 文件或目录上的元数据映射。建表时定义字段、分隔符、存储格式和路径，查询时 Hive 根据元数据读取 HDFS 数据并解析为表结构。

## 三、函数与窗口函数

### 12. Hive 常见函数可以分为哪几类？

常见包括单行函数、聚合函数、炸裂函数、窗口函数和自定义函数。单行函数如字符串、日期、条件函数；聚合函数如 `count`、`sum`；炸裂函数如 `explode`；窗口函数如 `row_number`、`rank`、`lag`；自定义函数包括 UDF、UDAF、UDTF。

### 13. `collect_list` 和 `collect_set` 有什么区别？

`collect_list` 会把分组内数据收集为数组并保留重复值；`collect_set` 会去重后收集为数组。需要保留明细重复值时用 `collect_list`，需要去重聚合时用 `collect_set`。

### 14. `explode` 和 `lateral view` 常用于什么场景？

它们常用于一行转多行，例如把数组、Map 或逗号分隔字段拆成多行再统计。`explode` 负责炸裂数据，`lateral view` 用于把炸裂结果与原表其他字段关联起来。

### 15. `row_number`、`rank`、`dense_rank` 有什么区别？

`row_number` 为每行生成连续序号，即使排序值相同也不并列；`rank` 遇到并列会跳号；`dense_rank` 遇到并列不跳号。TopN 且要求唯一名次常用 `row_number`，要求体现并列名次常用 `rank` 或 `dense_rank`。

### 16. `lag` 和 `lead` 适合解决什么问题？

`lag` 获取当前行之前第 N 行的值，`lead` 获取当前行之后第 N 行的值。它们常用于计算相邻访问间隔、连续登录、订单前后状态变化等问题。

### 17. 窗口聚合和普通聚合有什么区别？

普通聚合会把多行压缩成一行；窗口聚合是在保留明细行的基础上计算分组内累计、排名、滑动统计等结果。例如累计订单金额可以用 `sum(amount) over(partition by user_id order by dt rows between unbounded preceding and current row)`。

## 四、Hive SQL 高频案例

### 18. 如何求连续 3 天及以上满足条件的用户？

常用等差数列法：先按用户和日期过滤满足条件的数据，再用 `row_number` 排序，计算 `date_sub(dt, rn)` 作为分组标识；连续日期减去连续序号后结果相同，最后按用户和标识分组统计天数。

### 19. 如何按访问间隔划分用户 Session？

先用 `lag(ts)` 取上一条访问时间，再计算当前时间与上一条时间差。如果间隔超过阈值，如 60 秒，则标记为新会话；最后对标记做窗口累计，生成 session id。

### 20. 如何统计允许间隔一天的最大连续登录天数？

可先按用户排序，用 `lag` 计算相邻登录日期差。如果日期差大于允许间隔则开启新分组，再通过窗口累计生成分组号，最后按用户和分组统计连续天数并取最大值。

### 21. 如何处理日期区间交叉或合并问题？

常用窗口函数维护当前行之前的最大结束日期：`max(edt) over(partition by id order by stt rows between unbounded preceding and 1 preceding)`。如果当前开始日期大于历史最大结束日期，说明开启新区间；否则属于已有重叠区间。

### 22. 如何统计同时在线峰值？

把上线时间转换为 `+1` 事件，下线时间转换为 `-1` 事件，按时间排序后做窗口累加，累加值就是每个时刻在线人数，取最大值即峰值。

### 23. Hive 中 TopN 通常怎么写？

按分组字段使用 `row_number() over(partition by 分组字段 order by 指标 desc)` 生成排名，再在外层过滤 `rn <= N`。如果需要并列名次，可使用 `rank` 或 `dense_rank`。

## 五、执行计划与性能优化

### 24. Explain 有什么作用？

`explain` 用于查看 Hive SQL 的执行计划，包括 Stage、Operator Tree、TableScan、Filter、Group By、Join、Reduce Output、File Output 等信息。它可以帮助判断是否发生全表扫描、是否触发 Map Join、Shuffle 阶段是否过多等。

### 25. Map-Side 聚合的作用是什么？

Map-Side 聚合会先在 Map 端对数据做局部聚合，减少进入 Shuffle 的数据量，从而降低网络传输和 Reduce 压力。相关参数如 `hive.map.aggr=true`。

### 26. Common Join 和 Map Join 有什么区别？

Common Join 通常需要 Shuffle，把相同 Join Key 的数据拉到同一个 Reduce 端完成 Join。Map Join 会把小表加载到内存，在 Map 端直接与大表 Join，适合小表足够小的场景，可减少 Shuffle。

### 27. Bucket Map Join 和 SMB Join 适合什么场景？

当两张表按 Join Key 分桶，且桶数满足倍数关系时，可以使用 Bucket Map Join 减少 Shuffle。如果两表都按 Join Key 分桶且桶内有序，可使用 Sort Merge Bucket Map Join，进一步提升大表 Join 性能。

### 28. Group By 数据倾斜怎么处理？

可以开启 `hive.groupby.skewindata=true`，让 Hive 通过两阶段聚合缓解热点 Key 压力；也可以在业务层过滤异常 Key、对热点 Key 单独处理、加随机前缀打散后再二次聚合。

### 29. Join 数据倾斜怎么处理？

常见方式包括过滤无效或空 Key、小表使用 Map Join、热点 Key 单独拆分处理、给倾斜 Key 加随机前缀打散、开启 `hive.optimize.skewjoin=true` 等。核心目标是避免大量相同 Key 集中到单个 Reduce。

### 30. Hive SQL 优化的一般思路是什么？

先用分区裁剪和列裁剪减少输入数据，再减少 Shuffle 和 Reduce 数量；小表优先 Map Join，大表 Join 关注分桶和倾斜；聚合关注 Map 端预聚合；复杂 SQL 用 Explain 定位瓶颈，并结合资源参数和文件格式优化。
