# Hadoop 高频面试题

## 整理依据

基于本目录 Hadoop 环境搭建、集群配置、常用 Shell、生产调优手册、源码解析等笔记整理，重点覆盖 Hadoop 生态定位、集群部署、核心配置、生产调优、常见故障与源码原理。

## 一、基础概念与生态定位

### 1. Hadoop 主要由哪些核心模块组成？

Hadoop 常见核心模块包括 HDFS、MapReduce、YARN 和 Common。HDFS 负责分布式存储，MapReduce 负责离线计算，YARN 负责资源调度，Common 提供通用工具、RPC、配置加载等基础能力。

### 2. Hadoop 常见配置文件有哪些？

Hadoop 3.x 常见配置文件包括 `core-site.xml`、`hdfs-site.xml`、`yarn-site.xml`、`mapred-site.xml` 和 `workers`。Hadoop 2.x 中从节点列表通常是 `slaves`。

### 3. `core-site.xml`、`hdfs-site.xml`、`yarn-site.xml`、`mapred-site.xml` 分别配置什么？

`core-site.xml` 配置 Hadoop 通用参数，如 `fs.defaultFS`、临时目录等；`hdfs-site.xml` 配置 HDFS 存储、NameNode、DataNode、副本等；`yarn-site.xml` 配置 ResourceManager、NodeManager、调度器和日志聚集；`mapred-site.xml` 配置 MapReduce 运行框架、历史服务器等。

### 4. Hadoop 常用端口有哪些？

Hadoop 3.x 中 NameNode 内部通信常用 8020/9820，NameNode Web UI 是 9870，YARN Web UI 是 8088，历史服务器 Web UI 是 19888。Hadoop 2.x NameNode Web UI 常用 50070。

## 二、集群搭建与运维

### 5. 完全分布式 Hadoop 集群搭建前需要做哪些基础准备？

通常需要准备 Linux 环境、安装 JDK、配置主机名和 IP 映射、关闭防火墙、配置 SSH 免密登录、规划 `/opt/module` 和 `/opt/software` 等目录、配置环境变量，并保证集群时间同步。

### 6. 为什么要配置 SSH 免密登录？

Hadoop 的 `start-dfs.sh`、`start-yarn.sh` 等群起脚本需要通过 SSH 登录其他节点启动进程。没有免密登录时，批量启停会频繁要求输入密码，自动化运维无法顺利执行。

### 7. 第一次启动 HDFS 为什么需要格式化 NameNode？

格式化 NameNode 会初始化 HDFS 元数据目录，生成集群 ID、FsImage、VERSION 等基础元数据文件。只有第一次搭建集群或确认清空旧数据时才应格式化。

### 8. 生产环境为什么不能随意重新格式化 NameNode？

重新格式化会生成新的集群 ID，可能导致 NameNode 和 DataNode 的集群 ID 不一致，旧数据无法被识别。若确需重建，应先停止相关进程，明确备份或清理所有节点的 data/logs 目录，避免误删生产数据。

### 9. Hadoop 集群常用启停方式有哪些？

整体启停 HDFS 使用 `start-dfs.sh`、`stop-dfs.sh`；整体启停 YARN 使用 `start-yarn.sh`、`stop-yarn.sh`。单进程启停可用 `hdfs --daemon start/stop namenode|datanode|secondarynamenode`、`yarn --daemon start/stop resourcemanager|nodemanager`。

### 10. 为什么生产环境要配置历史服务器和日志聚集？

历史服务器用于保存和查看已完成 MapReduce 任务的信息；日志聚集会把各 NodeManager 上的任务日志集中到 HDFS，并通过历史服务器查看，便于任务失败后的排查和审计。

### 11. Hadoop 集群为什么要做时间同步？

分布式集群依赖多个节点协同工作，时间不一致会影响日志排查、任务调度、Kerberos 认证、数据时间戳和监控告警。无法连接外网时通常选择一台节点作为内部 NTP 时间服务器。

## 三、核心原理与源码理解

### 12. Hadoop 配置参数的加载顺序是什么？

客户端代码中显式设置的参数优先级最高，其次是 ClassPath 下的用户自定义配置文件，再是集群自定义配置文件，最后是默认配置文件。`Configuration` 会加载默认配置和 `*-site.xml` 配置，供 `FileSystem.get()` 等组件使用。

### 13. Hadoop RPC 的基本思想是什么？

Hadoop RPC 通过协议接口约定客户端和服务端的方法，客户端调用接口方法，实际执行发生在服务端。NameNode、DataNode、客户端之间大量通信都依赖 RPC。

### 14. NameNode 启动时主要做哪些事情？

NameNode 启动时会解析参数，初始化 HTTP 服务，加载 FsImage 和 Edits 到内存，创建 RPC 服务端，检查元数据目录资源，启动块管理、安全模式和指标等公共服务。

### 15. NameNode 为什么会进入安全模式？

NameNode 启动时需要等待 DataNode 上报块信息，当满足最小安全块比例和最小可用 DataNode 等条件后才退出安全模式。安全模式期间通常不允许写入，用于防止块信息未恢复完成时发生错误操作。

### 16. NameNode 如何判断 DataNode 已经失联？

DataNode 会周期性向 NameNode 发送心跳。NameNode 通过心跳检查线程判断 `lastUpdate` 是否超过过期阈值，默认心跳间隔约 3 秒，失联判定时间常见约为 10 分钟 30 秒。

## 四、生产调优与故障排查

### 17. NameNode 内存如何估算？

NameNode 元数据主要存放在内存中，课程笔记中按每个文件块约 150Byte 粗略估算。文件数和块数越多，NameNode 内存压力越大，因此小文件过多会显著增加元数据压力。

### 18. Hadoop 3.x 中 NameNode 内存如何配置？

可在 `hadoop-env.sh` 中通过 `HADOOP_NAMENODE_OPTS=-Xmx...` 设置 NameNode 最大堆内存。Hadoop 3.x 支持根据机器内存动态分配，但生产中仍建议结合文件块规模明确配置。

### 19. 小文件过多会带来什么问题？

小文件会导致 NameNode 元数据膨胀，增加内存占用和 RPC 压力；MapReduce 读取时还可能产生大量 MapTask，调度开销远大于计算开销。常见治理方式是合并小文件、使用 SequenceFile/Parquet/ORC 或 CombineTextInputFormat。

### 20. 集群启动后发现某些进程没起来，排查思路是什么？

先用 `jps` 或自定义 `jpsall` 查看各节点进程；再查看对应服务日志；检查配置文件是否分发一致、环境变量是否生效、主机名解析是否正确、端口是否冲突、SSH 免密和权限是否正常。

### 21. NameNode 由于磁盘空间不足无法启动的原因是什么？

NameNode 启动时会通过资源检查确认 FsImage、Edits 等元数据目录是否有足够空间，默认会保留一定磁盘空间阈值。如果必需目录空间不足，资源检查失败，NameNode 可能进入异常状态或无法正常服务。

### 22. Hadoop 生产调优通常关注哪些方面？

常见关注点包括 NameNode 内存、DataNode 磁盘和心跳、HDFS 副本与块大小、YARN 容器资源、MapReduce Shuffle 缓冲、日志聚集、历史服务器、小文件治理、节点故障和数据均衡等。
