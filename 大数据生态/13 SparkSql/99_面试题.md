# SparkSQL 高频面试题

## 整理依据

基于本目录 SparkSQL 知识点、常用知识及既有面试题整理，重点覆盖 SparkSQL 概念、DataFrame/DataSet/RDD、SparkSession、SQL/DSL、数据源读写、Hive 集成、自定义函数和结构化数据处理实践。

## 一、基础概念

### 1. SparkSQL 是什么？

SparkSQL 是 Spark 用于处理结构化数据的模块，提供 DataFrame 和 DataSet 两个核心抽象，同时支持 SQL 查询、DSL 编程、Hive 兼容、JDBC/ODBC 访问和多种数据源读写。

### 2. SparkSQL 相比 Hive 有哪些优势？

Hive 早期主要基于 MapReduce，执行过程中大量中间结果落盘，I/O 开销大；SparkSQL 基于 Spark 引擎，可利用内存计算、Catalyst 优化器、列式存储、代码生成等能力，通常执行效率更高，并且更容易与 Spark 生态整合。

### 3. SparkSQL 的主要特点有哪些？

主要包括易整合 SQL 与 Spark 编程、统一的数据访问方式、兼容 Hive、支持 JDBC/ODBC 标准连接、支持 DataFrame/DataSet 高层 API、可通过 Catalyst 对执行计划进行优化。

### 4. SparkSQL 为什么能比 RDD API 更容易优化？

DataFrame/DataSet 带有 schema，SparkSQL 能理解字段名、类型、表达式、过滤条件和聚合关系，从而进行列裁剪、谓词下推、逻辑计划优化和物理计划选择；RDD 只知道对象类型，优化空间较小。

## 二、RDD、DataFrame、DataSet

### 5. DataFrame 是什么？

DataFrame 是带 schema 的分布式数据集，类似数据库二维表。它底层仍基于 RDD，但每一列有名称和类型，SparkSQL 可以基于结构信息进行查询优化。DataFrame 本质上是 `Dataset[Row]`。

### 6. DataSet 是什么？

DataSet 是 Spark 1.6 引入的强类型分布式数据集，结合了 RDD 的类型安全和面向对象编程风格，以及 SparkSQL 的查询优化能力。通常通过 case class 或 JavaBean 定义数据结构。

### 7. RDD、DataFrame、DataSet 的共同点是什么？

三者都是 Spark 平台下的分布式弹性数据集；都具有惰性执行机制；都有分区概念；都可以进行 filter、排序等操作；只有遇到 Action 或输出操作时才会真正触发计算。

### 8. RDD、DataFrame、DataSet 的主要区别是什么？

RDD 编译期类型安全，适合底层复杂逻辑，但缺少 SparkSQL 优化；DataFrame 有 schema 和 SQL 优化，但行类型为 Row，字段访问需要解析；DataSet 强类型，既支持类型安全，又能利用 SparkSQL 优化。

### 9. DataFrame 为什么反序列化成本更低？

DataFrame 有 schema，Spark 可以只反序列化需要的数据字段，而不必像 RDD 一样反序列化完整对象结构和内容；配合 off-heap 与编码机制，可减少 GC 和序列化开销。

### 10. DataSet 中 Encoder 的作用是什么？

Encoder 负责 JVM 对象与 Spark 内部二进制格式之间的高效转换。它使 DataSet 既保留强类型，又能按需读取字段并与 off-heap 内存交互，减少不必要的反序列化。

### 11. RDD、DataFrame、DataSet 如何相互转换？

RDD 转 DataFrame 常用 `toDF`，RDD 转 DataSet 常用 `toDS`；DataFrame 转 DataSet 使用 `as[类型]`；DataSet 转 DataFrame 使用 `toDF`；DataFrame/DataSet 转 RDD 使用 `.rdd`。Scala 中通常需要先导入 `import spark.implicits._`。

### 12. DataFrame 转 RDD 后数据类型是什么？

DataFrame 转 RDD 后类型是 `RDD[Row]`。Row 中字段可按下标访问，也可使用 `getAs[类型](字段名)` 获取。由于 Row 不是业务对象，字段访问需要注意类型和顺序。

## 三、SparkSession 与编程方式

### 13. SparkSession 的作用是什么？

SparkSession 是 SparkSQL 的统一入口，整合了早期 SQLContext 和 HiveContext。它内部封装 SparkContext，可用于创建 DataFrame、执行 SQL、读取数据源、注册临时视图和启用 Hive 支持。

### 14. SQL 风格和 DSL 风格有什么区别？

SQL 风格需要先创建临时视图或全局视图，再通过 `spark.sql()` 执行 SQL；DSL 风格直接调用 DataFrame API，如 select、filter、groupBy，不必注册视图，更适合与代码逻辑结合。

### 15. 临时视图和全局临时视图有什么区别？

普通临时视图只在当前 SparkSession 中有效；全局临时视图在应用级别有效，跨 Session 可访问，访问时需要使用 `global_temp.视图名`。

### 16. 为什么要导入 `spark.implicits._`？

它提供 RDD、Seq 与 DataFrame/DataSet 之间转换所需的隐式转换，以及 `$"col"` 等列表达式能力。必须在 SparkSession 创建之后导入，因为 `spark` 是对象名，不是包名。

## 四、数据源读写

### 17. SparkSQL 支持哪些常见数据源？

常见包括 json、csv、parquet、orc、jdbc、text、Hive 表等。可通过 `spark.read.format(...).option(...).load(...)` 统一读取，通过 `df.write.format(...).mode(...).save(...)` 统一写出。

### 18. SparkSQL 默认的数据源格式是什么？

默认数据源格式是 Parquet。可以通过配置 `spark.sql.sources.default` 修改默认格式。Parquet 是列式存储格式，适合结构化、嵌套数据和分析型查询。

### 19. Spark 读取 JSON 文件有什么注意点？

SparkSQL 读取 JSON 时，通常要求文件中每一行是一个完整 JSON 字符串，而不是传统意义上的多行整体 JSON 文档。Spark 会自动推断 schema，但生产中建议显式指定 schema 以减少推断开销和类型不稳定问题。

### 20. SaveMode 中 append 和 overwrite 有什么区别？

append 是在已有路径或表中追加写入；overwrite 是覆盖已有数据或分区。需要注意 SaveMode 本身不是加锁的，也不是严格原子操作，生产中要结合表格式、分区策略和任务幂等性设计。

### 21. SparkSQL 如何通过 JDBC 读写 MySQL？

读取可使用 `spark.read.format("jdbc")` 并指定 url、driver、user、password、dbtable；写入可使用 `df.write.format("jdbc")` 或 `.jdbc()`，并设置 SaveMode。生产中还需考虑分区读取、批量写入、连接数和事务压力。

### 22. SparkSQL 连接外部 Hive 需要哪些准备？

通常需要把 `hive-site.xml` 放到 Spark conf 或项目 resources 中，把 MySQL 驱动放到 jars 目录，若访问 HDFS 还需 core-site.xml、hdfs-site.xml。代码中需使用 `enableHiveSupport()` 启用 Hive 支持。

## 五、函数、优化与实战

### 23. UDF 和 UDAF 分别是什么？

UDF 是用户自定义标量函数，一行输入返回一个值；UDAF 是用户自定义聚合函数，多行输入聚合为一个值。Spark 3.0 后更推荐使用强类型 `Aggregator` 或 `functions.udaf` 实现聚合逻辑。

### 24. 弱类型 UDAF 和强类型 Aggregator 有什么区别？

弱类型 UDAF 通常基于 Row、StructType、MutableAggregationBuffer，类型检查较弱；强类型 Aggregator 定义输入类型、缓冲类型和输出类型，类型安全更好，并通过 Encoder 进行编码转换。

### 25. SparkSQL 的 Catalyst 优化器能做什么？

Catalyst 会对 SQL 或 DataFrame 操作生成逻辑计划，并进行规则优化，如谓词下推、列裁剪、常量折叠、表达式简化、Join 重排等，再生成物理计划并选择执行策略。

### 26. 什么是谓词下推？为什么重要？

谓词下推是把 filter 条件尽量下推到数据源扫描或 Join 之前执行，减少后续处理的数据量。对 Parquet、ORC、JDBC 等数据源尤其重要，可以减少 I/O、网络传输和 Shuffle 数据量。

### 27. SparkSQL 中 `coalesce(field1, field2, ...)` 函数有什么作用？

它会从左到右返回第一个非 null 的字段值；如果所有字段都是 null，可使用最后一个默认值作为兜底。常用于多字段补全、默认值处理和空值兼容。

### 28. SparkSQL 中如何生成随机数？

可以使用 `rand()` 生成 `[0,1)` 的随机小数。若要生成指定范围，可用 `rand() * (max - min + 1) + min`，再按需要取整或转换类型。

### 29. 各区域热门商品 Top3 这类需求如何实现？

一般先过滤点击行为，再关联城市表、商品表补全维度；按地区和商品分组统计点击次数；使用窗口函数或排序取各地区 Top3；城市占比等复杂备注可通过自定义 UDAF 或聚合逻辑实现。

### 30. SparkSQL 生产调优常见方向有哪些？

常见方向包括：减少扫描字段和数据量；合理使用分区和谓词下推；控制 `spark.sql.shuffle.partitions`；避免大宽表无条件 Join；小表使用广播 Join；处理数据倾斜；缓存复用中间结果；避免过多小文件；复杂 SQL 拆分执行。
