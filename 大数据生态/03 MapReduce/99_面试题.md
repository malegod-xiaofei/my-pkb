# MapReduce 高频面试题

## 整理依据

基于本目录 MapReduce 知识点、MapReduce 常用、编程规范、序列化、InputFormat、Shuffle、Partition、排序、Combiner、OutputFormat 等笔记整理，重点覆盖框架原理、数据倾斜、调优和开发场景题。

## 一、基础概念与编程模型

### 1. MapReduce 是什么？

MapReduce 是一个分布式离线计算编程框架，用于将用户业务逻辑和 Hadoop 默认组件整合成可在集群上并发运行的分布式程序，适合 PB 级海量数据离线处理。

### 2. MapReduce 的优缺点是什么？

优点是易于编程、扩展性好、容错性强，适合海量离线计算。缺点是不擅长实时计算、流式计算和复杂 DAG 计算，因为多个 MR 作业之间通常要落盘，磁盘 IO 开销大。

### 3. 一个 MapReduce 程序在运行时有哪些进程？

主要有 MrAppMaster、MapTask、ReduceTask。MrAppMaster 负责作业调度和状态协调；MapTask 负责 Map 阶段数据处理；ReduceTask 负责 Reduce 阶段聚合输出。

### 4. MapReduce 编程规范包含哪几部分？

通常包括 Mapper、Reducer 和 Driver 三部分。Mapper 处理输入数据并输出中间 KV；Reducer 对同 key 的数据聚合；Driver 创建 Job、设置输入输出路径、设置 Mapper/Reducer、KV 类型、分区、排序等参数并提交任务。

### 5. Mapper 和 Reducer 中常见生命周期方法有哪些？

Mapper 常见方法包括 `setup()`、`map()`、`cleanup()`；Reducer 常见方法包括 `setup()`、`reduce()`、`cleanup()`。`setup` 用于初始化资源，核心业务在 `map` 或 `reduce` 中，`cleanup` 用于关闭资源。

## 二、序列化与输入输出

### 6. Hadoop 为什么不用 Java 原生序列化？

Java 原生序列化较重，会携带较多类信息和校验信息，网络传输效率低。Hadoop 使用 Writable 序列化，更紧凑、快速，适合分布式计算中的大量 KV 传输。

### 7. 自定义 Bean 作为 MapReduce 值对象时需要注意什么？

需要实现 `Writable`，提供空参构造，重写 `write()` 和 `readFields()`，且序列化与反序列化字段顺序必须完全一致。若要作为 key 参与排序，还要实现 `WritableComparable` 并重写 `compareTo()`。

### 8. 默认 TextInputFormat 的 KV 是什么？

默认 `TextInputFormat` 按行读取文本。Key 是该行在文件中的起始字节偏移量，类型通常是 `LongWritable`；Value 是当前行内容，类型是 `Text`。

### 9. InputFormat 的作用是什么？

InputFormat 负责决定输入数据如何切片以及如何读取记录。常见实现包括 TextInputFormat、KeyValueTextInputFormat、NLineInputFormat、CombineTextInputFormat 和自定义 InputFormat。

### 10. OutputFormat 的作用是什么？

OutputFormat 负责定义最终结果如何输出。默认是 TextOutputFormat，按行输出文本。需要输出到多个文件、过滤日志或自定义存储格式时，可自定义 OutputFormat 和 RecordWriter。

## 三、切片与并行度

### 11. MapTask 数量由哪些因素决定？

主要由输入文件数量、文件大小、BlockSize、InputFormat 切片规则以及文件是否可切分决定。一个切片通常对应一个 MapTask。

### 12. Block 和 Split 有什么区别？

Block 是 HDFS 物理存储单位，实际数据按块存储；Split 是 MapReduce 的逻辑计算切片，不会真正切割磁盘文件。Split 决定 MapTask 并行度。

### 13. MapTask 是越多越好吗？

不是。MapTask 过少会并行度不足，过多会带来任务启动、调度、JVM 和 Shuffle 开销。小文件场景若每个文件一个 MapTask，性能反而很差。

### 14. CombineTextInputFormat 解决什么问题？

它主要解决大量小文件导致 MapTask 过多的问题。CombineTextInputFormat 会将多个小文件在逻辑上合并到一个或多个切片中，让一个 MapTask 处理多个小文件，降低调度开销。

### 15. FileInputFormat 切片时为什么有 1.1 倍判断？

为了避免产生过小的尾部切片。当剩余数据小于切片大小的约 1.1 倍时，通常不再单独切分，而是合并到当前切片中，减少小切片带来的额外任务开销。

## 四、Shuffle、分区与排序

### 16. Shuffle 是什么？

Shuffle 是 Map 方法之后、Reduce 方法之前的数据处理过程，包括 Map 端缓冲、溢写、分区、排序、合并，Reduce 端拉取、合并、分组等步骤，是 MapReduce 性能调优重点。

### 17. MapReduce Shuffle 的主要流程是什么？

MapTask 将输出 KV 写入内存缓冲区；缓冲区达到阈值后溢写到磁盘，溢写时进行分区和排序；多个溢写文件合并；ReduceTask 按分区从各 MapTask 拉取数据；Reduce 端归并排序并按 key 分组，最后调用 reduce 方法。

### 18. Shuffle 缓冲区大小对性能有什么影响？

缓冲区越大，Map 端溢写次数可能越少，磁盘 IO 和合并开销降低。常见参数是 `mapreduce.task.io.sort.mb`，默认约 100M，需要结合内存资源和任务规模调整。

### 19. 默认分区规则是什么？

默认分区器是 HashPartitioner，规则通常是 `(key.hashCode() & Integer.MAX_VALUE) % numReduceTasks`。相同 key 会进入同一个 ReduceTask。

### 20. 自定义分区时需要注意什么？

需要继承 Partitioner 并重写 `getPartition()`，同时在 Driver 中设置分区类和 ReduceTask 数量。分区号必须小于 ReduceTask 数量，否则会报错；ReduceTask 数量少于分区数会导致部分分区无法输出到独立文件。

### 21. MapReduce 中有哪些常见排序？

常见有部分排序、全排序和二次排序。部分排序是每个 Reduce 输出文件内部有序；全排序通常设置一个 ReduceTask；二次排序可通过自定义组合 key、排序规则和分组规则实现。

### 22. Combiner 的作用和使用前提是什么？

Combiner 在 Map 端做局部聚合，减少网络传输和 Reduce 压力。前提是不能改变最终业务结果，求和、计数通常可以，平均值这类依赖全局数据的逻辑不能直接使用普通 Combiner。

## 五、调优与生产场景

### 23. 如何优化 MapReduce 小文件场景？

可使用 CombineTextInputFormat 合并输入切片；从源头合并小文件；使用 SequenceFile、ORC、Parquet 等存储格式；减少不必要的 MapTask，避免调度开销超过计算开销。

### 24. MapReduce 数据倾斜如何处理？

先定位倾斜 key 和倾斜阶段。常见措施包括合理设计分区器、对热点 key 加随机前缀做两阶段聚合、启用 Combiner、提高 ReduceTask 数量、过滤异常数据或单独处理热点 key。

### 25. 如何动态给 MapReduce 作业传递参数？

可使用 `-Dkey=value` 传递参数，例如 `yarn jar xxx.jar Main -Dk1=v1 /input /output`。程序中可通过 Configuration 获取参数；若自定义 Driver 需要正确解析 `-D`，可使用 GenericOptionsParser 或 Tool 接口。
