# Scala 高频面试题

## 整理依据

基于本目录 Scala 基础与进阶笔记整理，重点覆盖 Scala 与 JVM、变量与类型、控制结构、函数式编程、集合体系、常用高阶函数、WordCount、模式匹配、样例类和泛型等高频知识点。

## 一、基础与 JVM 关系

### 1. Scala 是什么？和 Java 有什么关系？

Scala 是运行在 JVM 上的多范式编程语言，同时支持面向对象和函数式编程。Scala 源码会编译成字节码运行在 JVM 上，可以与 Java 类库互操作。

### 2. Scala 中 `object` 和 `class` 有什么区别？

`class` 定义普通类，需要实例化后使用；`object` 定义单例对象，天然只有一个实例。Scala 没有 Java 风格的 `static`，通常用 `object` 存放静态方法、常量和程序入口。

### 3. Scala 程序入口通常怎么写？

常见方式是在单例对象中定义 `main(args: Array[String]): Unit` 方法，或继承 `App`。课程中常见写法是 `object Xxx { def main(args: Array[String]): Unit = { ... } }`。

### 4. Scala 为什么说一切运算符本质上都是方法？

Scala 中很多符号运算本质是方法调用，例如 `1 + 2` 等价于 `1.+(2)`。这体现了 Scala 面向对象设计，也解释了为什么方法名可以是符号。

## 二、变量、类型与表达式

### 5. `var` 和 `val` 有什么区别？

`var` 定义变量，引用可以重新赋值；`val` 定义常量，引用不可重新赋值。若 `val` 指向的是可变对象，对象内部状态仍可能变化，但该引用不能指向新对象。

### 6. Scala 类型推导有什么特点？

Scala 是强类型语言，但支持类型推导。声明变量或函数返回值时，编译器能根据右侧表达式推导类型；复杂场景或对外 API 通常建议显式声明类型。

### 7. Scala 的 `Unit`、`Null`、`Nothing` 分别表示什么？

`Unit` 类似 Java 的 `void`，表示无有意义返回值；`Null` 是引用类型的子类型，其唯一实例是 `null`；`Nothing` 是所有类型的子类型，常用于异常、无限循环或不会正常返回的方法。

### 8. Scala 中 `if` 和 Java 中有什么不同？

Scala 的 `if` 是表达式，有返回值，可以直接赋给变量。不同分支返回类型不一致时，结果类型会向共同父类型推导，例如 `Any`。

### 9. Scala 字符串插值有哪些常见方式？

常见有 `s` 插值和多行字符串。`s"name=$name"` 可直接引用变量，`${}` 可写表达式；多行字符串常结合 `stripMargin` 去除边界符，适合书写 SQL。

### 10. Scala 自动类型转换和强制类型转换要注意什么？

低精度数值可自动转换为高精度数值。高精度转低精度需要强制转换，如 `toInt`，可能发生精度丢失。String 与基本类型互转常用 `toInt`、`toDouble`、`toString` 等方法。

## 三、流程控制与函数式编程

### 11. Scala 的 `for` 推导式常见用法有哪些？

`for` 可遍历集合或范围，如 `1 to 10`、`1 until 10`；可添加守卫条件；也可使用 `yield` 生成新集合。`for-yield` 是函数式风格中常见的数据转换方式。

### 12. Scala 中 `while` 为什么不如 `for` 和集合函数常用？

`while` 偏命令式，返回值通常是 `Unit`，不利于表达式编程。Scala 更鼓励使用 `for`、`map`、`filter`、`fold` 等表达式和高阶函数处理集合。

### 13. 什么是高阶函数？

高阶函数是指可以接收函数作为参数，或返回函数作为结果的函数。Scala 集合中的 `map`、`flatMap`、`filter`、`reduce`、`fold` 等都是典型高阶函数。

### 14. `map`、`flatMap`、`flatten` 有什么区别？

`map` 对集合中每个元素做一对一转换；`flatten` 把嵌套集合压平一层；`flatMap` 等价于先 `map` 再 `flatten`，常用于切分单词、展开数组等一对多转换。

## 四、集合体系

### 15. Scala 集合分为哪几大类？

Scala 集合主要包括 Seq、Set、Map 三大类，并统一继承 Iterable。集合又分为不可变集合和可变集合，默认通常使用不可变集合。

### 16. 不可变集合和可变集合有什么区别？

不可变集合修改后会返回新集合，原集合不变；可变集合可在原对象上直接增删改。不可变集合更安全，适合函数式编程；可变集合在局部高性能场景下也常用。

### 17. Array 和 ArrayBuffer 有什么区别？

Array 是定长数组，底层接近 Java 数组；ArrayBuffer 是可变长数组，支持动态追加和删除元素。需要固定长度和高性能访问时用 Array，需要频繁增删时用 ArrayBuffer。

### 18. List 和 ListBuffer 有什么区别？

List 是不可变链表，适合递归和头部操作；ListBuffer 是可变列表，适合频繁追加或构建后再转为 List。List 常用 `::` 追加到头部，`:::` 拼接列表。

### 19. Set 和 Map 在 Scala 中有什么特点？

Set 表示不重复元素集合，Map 表示键值对集合。不可变 Map 添加或删除元素会返回新 Map；访问不存在 Key 时可使用 `getOrElse` 提供默认值，避免直接取值异常。

### 20. Tuple 适合什么场景？

Tuple 用于临时组合多个不同类型的值，例如函数返回多个结果、键值中间结果等。访问元素可使用 `_1`、`_2` 等，但字段含义不明显时应优先定义样例类。

## 五、集合计算与案例

### 21. `reduce`、`fold`、`foldLeft`、`foldRight` 有什么区别？

`reduce` 使用集合自身元素聚合，没有外部初始值；`fold` 有初始值；`foldLeft` 从左向右聚合；`foldRight` 从右向左聚合。左右方向不同会影响非交换运算的结果。

### 22. `groupBy` 常用于什么场景？

`groupBy` 按指定规则把集合元素分组，返回 Map。常用于按字段分类、奇偶分组、单词分组统计等场景。

### 23. `sortBy` 和 `sortWith` 有什么区别？

`sortBy` 按某个字段或表达式排序，写法简洁；`sortWith` 接收两个元素的比较函数，适合复杂排序规则。

### 24. Scala WordCount 的核心思路是什么？

典型流程是：读取行数据，使用 `flatMap(_.split(" "))` 切分单词，再用 `groupBy` 按单词分组，随后统计每组大小，最后按词频排序输出。

### 25. `sliding` 和 `zip` 常用于什么场景？

`sliding` 用于滑动窗口处理，例如相邻元素组合、连续区间统计；`zip` 用于把两个集合按位置组合成元组，常用于元素与索引或两个序列配对。

## 六、模式匹配与高级特性

### 26. Scala 模式匹配相比 Java `switch` 有什么优势？

Scala 模式匹配不仅能匹配常量，还能匹配类型、数组、列表、元组、对象、样例类，并支持模式守卫，表达能力更强，返回值也可作为表达式使用。

### 27. 什么是模式守卫？

模式守卫是在 `case` 后添加 `if` 条件，对匹配结果进一步过滤。例如 `case i: Int if i >= 0 => i` 可匹配非负整数。

### 28. 模式匹配中类型匹配要注意什么？

JVM 泛型存在类型擦除，模式匹配中 `List[Int]` 和 `List[String]` 在运行期可能无法准确区分。但数组类型在 JVM 中保留元素类型，匹配 `Array[Int]` 等相对特殊。

### 29. 样例类在模式匹配中有什么作用？

样例类会自动生成 `apply`、`unapply`、`toString`、`equals`、`hashCode` 等方法，适合不可变数据建模，并能在模式匹配中直接解构字段。

### 30. Scala 集合面试中最常见的考察点是什么？

常见考察点包括可变与不可变集合区别、List/Array/Map/Set/Tuple 使用、高阶函数 `map`/`flatMap`/`groupBy`/`reduce`/`fold`、WordCount 实现、模式匹配和泛型擦除。
