# Doris 高频面试题

## 整理依据

- `01_知识点.md`：Doris 简介、FE/BE/Broker 架构、编译安装、集群部署、扩缩容、表创建、分区分桶、数据模型、动态分区。
- `02_实战.md`：数据库版本查询。

## 一、基础与架构

### 1. Apache Doris 是什么，适合哪些场景？

Apache Doris 是现代化 MPP 分析型数据库，支持亚秒级查询和实时数据分析，适合固定历史报表、实时分析、交互式分析、探索式分析以及 PB 级 OLAP 查询场景。

### 2. Doris 的架构有什么特点？

Doris 架构简洁，核心只有 FE 和 BE 两类进程，不依赖外部组件即可运行，便于部署运维并支持线性扩展。通过 MySQL 协议对外提供访问能力。

### 3. FE 的职责是什么？

FE 负责存储和维护元数据，接收并解析 SQL 请求，生成查询计划，调度 BE 执行并返回结果。FE 还负责集群管理、权限、导入任务协调等工作。

### 4. BE 的职责是什么？

BE 负责物理数据存储和计算执行。它依据 FE 生成的物理计划分布式执行查询，同时负责 Tablet 副本、数据可靠性、Compaction 和本地数据读写。

### 5. Doris 中 FE 有哪些角色？

FE 包括 Leader、Follower 和 Observer。Leader/Follower 组成高可用元数据组，Leader 宕机后 Follower 可重新选主；Observer 用于扩展查询能力和元数据备份，但不参与写入和选举。

### 6. Broker 的作用是什么？

Broker 是独立无状态进程，封装外部文件系统接口，使 Doris 能读取 HDFS、S3、BOS 等远端存储中的文件，常用于 Broker Load 等导入场景。

## 二、部署与运维

### 7. Doris 生产部署有哪些注意事项？

生产建议 FE 和 BE 分开部署；FE 元数据目录单独指定磁盘，JVM 堆内存适当调大；BE 数据目录预留充足空间，按用户数据量、三副本和 Compaction 中间数据估算；所有节点关闭 Swap；多 FE 节点时钟偏差不能过大。

### 8. FE 高可用如何部署？

部署 3 个 Follower 可实现读写高可用，Follower 数量必须为奇数；也可增加 Observer 扩展读查询能力。新增 Follower/Observer 后，第一次启动需要指定 `--helper leader_host:edit_log_port`。

### 9. 为什么 Doris 三副本至少需要 3 台不同 IP 的机器？

Doris 副本分布以 IP 识别物理机，同一个 Tablet 的多个副本不能放在同一台物理机上。即使一台机器部署多个 BE，如果 IP 相同，副本数也只能视为 1。

### 10. BE 如何扩容和缩容？

扩容通过 `ALTER SYSTEM ADD BACKEND` 添加节点。缩容推荐使用 `ALTER SYSTEM DECOMMISSION BACKEND`，Doris 会先迁移该 BE 上的数据，迁移完成后再下线；不推荐 `DROP BACKEND`，因为会直接删除节点且数据不可恢复。

### 11. 如何判断 FE、BE、Broker 状态？

可通过 MySQL Client 连接 FE 后执行 `SHOW PROC '/frontends'`、`SHOW PROC '/backends'`、`SHOW PROC '/brokers'`。BE 的 `Alive=true` 表示节点存活，Decommission 和 TabletNum 可用于观察下线过程。

### 12. BE 下线 Decommission 失败的常见原因有哪些？

常见原因包括剩余 BE 存储空间不足、剩余机器数量不满足最小副本数、数据迁移无法完成等。此时 BE 会保持 `isDecommission=true`，可继续排查资源或执行取消下线命令。

## 三、表设计与数据划分

### 13. Doris 中 Partition、Bucket、Tablet 分别是什么？

Partition 是逻辑分区，常按时间或枚举值划分；Bucket 是分区内按 Hash 分桶；每个 Bucket 对应一个 Tablet，Tablet 是数据移动、复制和存储的最小逻辑单元。

### 14. Doris 支持哪些分区方式？

Doris 支持 Range 分区和 List 分区。Range 常用于时间列，便于按时间管理冷热数据和历史删除；List 用于枚举值分区，如城市、地区等。

### 15. Range 分区删除后为什么可能出现分区空洞？

使用 `VALUES LESS THAN` 定义分区时，分区范围不会因删除已有分区而自动重排。删除中间分区后，该时间范围不再属于任何分区，后续导入落入空洞范围的数据会失败。

### 16. 分桶列如何选择？

分桶列需要是 Key 列。多列分桶通常数据更均匀，适合大吞吐低并发查询；少数分桶列或单列分桶适合高并发点查，因为等值条件可能只扫描少量桶，减少查询间 IO 干扰。

### 17. 复合分区适合什么场景？

复合分区适合有时间维度、需要按周期删除历史数据、不同分区数据量差异较大或需要解决数据倾斜的场景。可以按时间分区，再按高区分度字段分桶。

### 18. `replication_num` 设置有什么注意事项？

`replication_num` 表示每个 Tablet 的副本数量，默认 3，建议使用奇数。最大副本数受集群不同 IP 数限制。小而更新不频繁的维度表可适当增加副本，提高 Join 本地命中概率。

### 19. `storage_medium` 和 `storage_cooldown_time` 的作用是什么？

`storage_medium` 指定初始存储介质为 SSD 或 HDD；`storage_cooldown_time` 指定数据从 SSD 迁移到 HDD 的时间。该机制用于冷热数据分层存储，但实际效果也受 BE 存储目录和 FE 严格校验参数影响。

## 四、数据模型

### 20. Doris 有哪些数据模型？

主要包括 Aggregate、Uniq 和 Duplicate。Aggregate 支持按 Key 预聚合；Uniq 用于主键唯一约束，本质类似 REPLACE 聚合；Duplicate 保存完整明细，不做聚合或去重。

### 21. Aggregate 模型适合什么场景？

Aggregate 模型适合固定报表和汇总分析，通过 Key 列作为维度，Value 列按 SUM、REPLACE、MAX、MIN 等聚合类型合并，减少存储量和聚合查询计算量。

### 22. Aggregate 模型的数据聚合发生在哪些阶段？

聚合可能发生在导入 ETL 阶段、BE 后台 Compaction 阶段和查询阶段。不同阶段聚合程度可能不同，但 Doris 会通过查询阶段聚合保证对外结果一致。

### 23. Aggregate 模型为什么对 `count()` 不友好？

Aggregate 模型对外展示聚合后的结果。为了得到正确 `count()`，Doris 可能需要读取所有 Key 列并在查询时聚合，成本较高。常见优化是在表中增加恒为 1 的计数字段，并使用 SUM 或 REPLACE 聚合，通过 `sum(count_col)` 模拟计数。

### 24. Uniq 模型和 Aggregate 模型是什么关系？

Uniq 模型用于保证 Key 唯一，本质上可用 Aggregate 模型中 Value 列的 REPLACE 聚合实现。它适合需要唯一主键约束的分析场景，但无法充分利用 SUM 等预聚合优势。

### 25. Duplicate 模型适合什么场景？

Duplicate 模型适合无主键、无预聚合需求的明细数据或 Ad-hoc 查询场景。它会完整保留导入数据，即使两行完全相同也保留；`DUPLICATE KEY` 主要表示底层排序列。

## 五、动态分区与实战

### 26. 动态分区解决什么问题？

动态分区用于表级分区生命周期管理，自动创建未来分区和删除过期分区，降低按天、按周、按月维护分区的人工成本，避免因忘记创建分区导致导入失败。

### 27. 动态分区有哪些关键参数？

常见参数包括 `dynamic_partition.enable`、`time_unit`、`start`、`end`、`prefix`、`buckets`、`replication_num`、`create_history_partition`、`history_partition_num`、`hot_partition_num`、`reserved_history_periods` 等。

### 28. 动态分区中 `start` 和 `end` 分别表示什么？

`start` 是相对当前时间的历史偏移，通常为负数，用于删除过期分区；`end` 是未来偏移，用于提前创建未来分区。例如按天分区，`start=-7`、`end=3` 表示保留最近 7 天并预建未来 3 天。

### 29. 动态分区丢失后会自动补齐吗？

如果 `dynamic_partition.start` 到当前时间之间的历史分区丢失，通常不会自动重新创建；当前时间到 `dynamic_partition.end` 之间的未来分区丢失，会被重新创建。

### 30. 如何查询 Doris 数据库版本？

可以执行：

```sql
SELECT @@version_comment;
```
