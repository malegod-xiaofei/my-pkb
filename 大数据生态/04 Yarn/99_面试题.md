# YARN 高频面试题

## 整理依据

基于本目录 YARN 知识点、常用 Shell 命令、工作机制、作业提交流程、调度器、核心参数、多队列配置和 Tool 接口等笔记整理，重点覆盖资源调度、生产队列、参数调优和故障排查。

## 一、基础架构与核心组件

### 1. YARN 是什么？

YARN 是 Hadoop 的资源调度平台，负责为 MapReduce、Spark、Flink 等计算程序分配集群资源，相当于分布式操作系统中的资源管理层。

### 2. YARN 有哪些核心组件？

核心组件包括 ResourceManager、NodeManager、ApplicationMaster 和 Container。ResourceManager 负责全局资源管理和调度；NodeManager 管理单节点资源和容器；ApplicationMaster 负责单个应用的任务申请、调度和状态协调；Container 是任务运行的资源抽象。

### 3. ResourceManager 和 NodeManager 分别负责什么？

ResourceManager 负责接收应用提交、资源调度、队列管理和应用状态维护。NodeManager 负责本节点资源上报、Container 启停、日志管理和任务运行监控。

### 4. ApplicationMaster 的作用是什么？

ApplicationMaster 是每个应用的管理者，负责向 ResourceManager 申请资源，协调 MapTask/ReduceTask 等任务运行，监控任务状态，并在应用结束后注销。

### 5. Container 是什么？

Container 是 YARN 对 CPU、内存等资源的封装单位。任务并不是直接运行在 NodeManager 上，而是在 NodeManager 分配和启动的 Container 中运行。

## 二、工作机制与作业提交流程

### 6. YARN 运行 MapReduce 作业的大致流程是什么？

客户端提交作业后，YarnRunner 向 RM 申请 Application；客户端上传 jar、切片信息和配置到 HDFS；RM 分配 Container 启动 MRAppMaster；MRAppMaster 向 RM 申请 MapTask 和 ReduceTask 资源；NodeManager 启动对应 Container 执行任务；任务完成后应用注销。

### 7. MapReduce 作业提交到 YARN 时会上传哪些资源？

通常会上传 jar 包、切片信息、作业配置文件等到 HDFS 的作业提交目录，供 ApplicationMaster 和任务 Container 下载运行。

### 8. 作业运行过程中客户端如何获取进度？

任务会向 ApplicationMaster 汇报进度、状态和 Counter。客户端通过周期性请求 ApplicationMaster 获取进度，作业完成后信息可由历史服务器保存和查看。

### 9. ReduceTask 什么时候开始申请资源？

课程流程中 MRAppMaster 等待所有 MapTask 运行完毕后，再向 ResourceManager 申请 Container 运行 ReduceTask。实际生产中也可受慢启动参数影响，但核心依赖 Map 输出数据。

## 三、调度器与队列

### 10. YARN 常见调度器有哪些？

常见调度器包括 FIFO、Capacity Scheduler 和 Fair Scheduler。Apache Hadoop 3.1.3 默认是 Capacity Scheduler，CDH 常见默认是 Fair Scheduler。

### 11. FIFO 调度器有什么特点？

FIFO 是单队列、按提交顺序先来先服务，优点是简单，缺点是不支持多队列和多租户隔离，生产环境较少使用。

### 12. Capacity Scheduler 的特点是什么？

容量调度器支持多队列、多用户和资源容量保障。每个队列可配置容量、最大容量、ACL、用户限制等，资源空闲时也可被其他队列借用，适合中小企业或资源隔离明确的场景。

### 13. Fair Scheduler 的特点是什么？

公平调度器支持多队列、多用户，目标是让队列或应用在一段时间内公平共享资源。并发要求高、业务多租户复杂的中大规模场景更常选择公平调度器。

### 14. 容量调度器和公平调度器的主要区别是什么？

二者都支持多队列和多用户。容量调度器更强调队列容量保障和先到任务优先满足；公平调度器更强调同队列或多队列任务公平享有资源，适合高并发资源竞争场景。

### 15. 生产环境为什么要配置多队列？

多队列可以做业务隔离、资源配额、权限控制和故障隔离，避免某个递归或异常任务耗尽全局资源。大促等特殊时期还可对重要队列做资源保障或降级。

### 16. 多队列通常按什么维度划分？

可以按计算框架划分，如 Hive、Spark、Flink、MR；也可以按业务模块划分，如登录、注册、购物车、下单、营销等。课程笔记更推荐按业务模块划分以降低业务风险。

### 17. 如何向指定队列提交 MapReduce 任务？

可在提交命令中指定 `-D mapreduce.job.queuename=队列名`，也可在 Driver 代码中通过 `conf.set("mapreduce.job.queuename", "hive")` 设置。

## 四、核心参数与生产调优

### 18. YARN 生产中常见资源参数有哪些？

常见参数包括 `yarn.nodemanager.resource.memory-mb`、`yarn.nodemanager.resource.cpu-vcores`、`yarn.scheduler.minimum-allocation-mb`、`yarn.scheduler.maximum-allocation-mb`、`yarn.scheduler.minimum-allocation-vcores`、`yarn.scheduler.maximum-allocation-vcores` 等。

### 19. 为什么生产中常关闭虚拟内存检查？

YARN 默认可能检查虚拟内存与物理内存比例，某些 Java 程序虚拟内存占用较高，容易被误杀。课程案例中通过 `yarn.nodemanager.vmem-check-enabled=false` 关闭虚拟内存检查，减少误杀风险。

### 20. 如何估算一个 MapReduce 作业需要多少 Container？

需要根据输入大小和切片大小估算 MapTask 数，再加上 ReduceTask 数和 ApplicationMaster。例如 1G 数据、128M 块大小约 8 个 MapTask，如果 1 个 ReduceTask，再加 1 个 MRAppMaster，总计约 10 个任务容器需求。

### 21. `yarn.resourcemanager.scheduler.client.thread-count` 如何设置？

该参数控制 RM 处理调度请求的线程数，默认较大。应结合集群 CPU 线程数和任务提交并发调整，不能盲目超过实际可用处理能力。

### 22. 队列配置变更后如何生效？

容量调度器队列配置可使用 `yarn rmadmin -refreshQueues` 刷新；部分参数变更可能需要重启 YARN。重启前应评估对正在运行任务的影响。

## 五、常用命令与排查

### 23. 常用 YARN 运维命令有哪些？

常用命令包括 `yarn application -list`、`yarn application -kill`、`yarn logs -applicationId`、`yarn node -list -all`、`yarn queue -status`、`yarn container -list`、`yarn applicationattempt -list`、`yarn rmadmin -refreshQueues`。

### 24. 如何查看失败任务日志？

先通过 `yarn application -list -appStates FAILED,KILLED,FINISHED` 找到 ApplicationId，再用 `yarn logs -applicationId <ApplicationId>` 查看应用日志；若需要具体容器，可加 `-containerId <ContainerId>`。

### 25. 为什么运行结束后 `yarn container -list` 可能看不到 Container？

`yarn container -list` 通常只能查看任务运行中的 Container。任务完成后 Container 信息可能不在 RM 中显示，需要通过历史服务器或日志聚集查看。
