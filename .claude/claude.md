## 笔记规范

- 笔记格式为 Markdown，在 Obsidian 中编辑
- 文件名使用中文或"序号 章节名"/"前缀 序号 章节名"格式（如 `机器学习 1 机器学习概述.md`、`c-01-MCP入门.md`）
- 课件原文件为 `.docx`，已在 `.gitignore` 中忽略，不上传 git
- 图片统一存放于对应目录的 `images/` 子文件夹（机器学习特殊：`机器学习算法图/`）
- 图片在 Markdown 中使用标准格式引用：`![图片N.png](images/图片N.png)`
- Markdown 正文中涉及 HTML 特殊标记（如 `<UNK>`、`<PAD>` 等）时，使用反引号包裹，避免 Obsidian 渲染异常

## 笔记整理说明

大模型核心技术三门课程的笔记均已从 `.docx` 课件整理为 Markdown，并按章节拆分为独立文件，整理策略：
- 标题层级：保留原文档一级/二级/三级/四五级标题
- 列表项：`参数列表样式` 转为 Markdown `- ` 列表
- 图片：从 docx 中提取（或引用已有）图片，存入对应 `images/` 文件夹，按 `图片N.png` 命名
- 章节拆分：以一级标题（`# 数字 章节名`）为边界，每章独立成一个 md 文件，原始整合文件保留

MCP 目录包含两套来源的笔记：
- 尚硅谷课程系列（`01~05`）：课程实战内容
- Claude Code 官方文档系列（`c-01~c-07`）：官方文档整理

## Git 工作流

- 主分支：`master`
- 推送方式：SSH（`git@github.com`）
- 提交粒度：按笔记章节或主题提交，commit message 使用中文描述

## .gitignore 规则说明

| 规则 | 原因 |
|------|------|
| `*.docx` | 课件原文件体积大，无需版本管理 |
| `.obsidian/workspace.json` | Obsidian 本地工作区状态，不同设备各异 |
| `.claude/settings.local.json` | 含本地权限配置和 SSH 路径 |
| `.claude/*.py` | 一次性处理脚本，用后即废 |
| `.claude/agents_*.md` / `.txt` | 临时中间产物 |
| `.claude/structure_*.txt` | 临时结构分析产物 |

## 注意事项

- `.claude/CLAUDE.md`（本文件）已纳入 git 跟踪，修改后需提交
- 新增临时脚本或草稿文件时，及时加入 `.gitignore`
- 大模型核心技术各课程的原始完整 md 文件与拆分后的章节文件共存于同一目录，原始文件作为备份保留
- 机器学习章节拆分文件命名前缀为 `机器学习 ` / `数学基础 `，与深度学习/NLP 的纯数字前缀格式不同
