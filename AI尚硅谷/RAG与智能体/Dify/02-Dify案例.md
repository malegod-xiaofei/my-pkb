# 配置模型

点击右上角用户账户 -> 设置 -> 模型供应商。

根据需要添加对应的模型供应商，也可以使用本地的Ollama，或者使用兼容OpenAI-API协议（`OpenAI-API-compatible`）。

根据实际情况，填写模型类型、Url、API key、模型名称。

  

# 插件安装

根据实际情况，为Dify安装对应插件。

在Dify工具页面（或者在Dify应用市场）下载安装插件，安装完成后，点击对应的插件，选择`去授权` ，填写对应应用的key完成授权。

  

# 工作流案例

## 创建工作流

创建应用类型为 `工作流`，配置应用名称、图标、描述。

创建应用之后，Dify会自动创建一个开始节点，但是不会自动创建结束节点，所以页面会有提示`检查清单：必须添加结束节点“。

## 配置开始节点

可以在`开始`节点中添加一个变量：

- 字段类型：`段落`
- 变量名：`target`
- 显示名称：`需求`
- 最大长度：`1024`
- 是否必填：勾选

  

## 配置搜索节点

为开始节点添加谷歌搜索：

选择添加`工具`->谷歌搜索（如果谷歌搜索没有授权，点击该节点时会提示授权）。

输入变量的查询语句中，输入`/`调取前面的`target`变量。

输出内容的样例：

```json
{
    "text": "",
    "files": [],
    "json": [
        {
            "organic_results": [
                {
                    "link": "http://xxxxx",
                    "snippet": "这是一个搜索内容",
                    "title": "这是一个标题"
                },
                {
                    "link": "http://xxxxxx",
                    "snippet": "xxxxx",
                    "title": "xxxx"
                }
            ]
        }
    ]
}
```

其中：`text`、`files`、`json`会分别赋值给对应的三个同名变量。

  

## 模拟搜索

如果没有`谷歌搜索`插件，或者没有连接外网功能，可以创建一个`代码执行`节点直接返回搜索结果进行模拟。

删除`谷歌搜索`节点，创建一个`代码执行`节点，名称命名为`模拟谷歌搜索`。

输入变量：无

代码：

```
def main() -> dict:
    return {
        "json": [
            {
                "organic_results": [
                    {
                        "link": "http://xxxxx/1",
                        "snippet": "这是一个搜索内容",
                        "title": "这是一个标题"
                    },
                    {
                        "link": "http://xxxxx/2",
                        "snippet": "这是第二个搜索内容",
                        "title": "这是第二个标题"
                    }
                ]
            }
        ]
    }
```

输出变量：

变量名：`json`

类型：`Array[Object]`

  

## 解析搜索结果

`谷歌搜索`工具输出的内容是一个JSON，需要使用代码进行解析，所以下一步添加节点 `代码执行`：

节点命名为`解析谷歌搜索`。

配置输入变量：根据实际情况，配置需要的输入变量的变量名，使用`/` 引用前面生成的变量。

变量名：`googleResult`

变量值：前面谷歌搜索的结果的`json`变量

编写python或 javascript 代码（代码的变量需要和上面定义的对应）。

```
def main(googleResult: list) -> dict:
    return {
        "result": googleResult[0]['organic_results']
    }
```

  

配置输出变量：输出的变量名、变量类型

变量名：`result`  
变量类型：`Array[Object]`

编写的代码可以`运行此步骤`进行测试运行。

  

## 迭代搜索结果

上面的代码解析出来的结果是一个json数组，所以需要加一个迭代的步骤进行遍历：下一个节点 `迭代`。迭代器命名为`迭代搜索记录`

迭代器中需要解析JSON，所以在`迭代`的块中，`迭代开始`的下一步增加一个`代码执行`，命名为`解析搜索结果中的标题`，

  

### 配置迭代块输入变量

输入：使用`/`引用前一步生成的`result`变量。

输出：先不配置，等迭代块的内容配置完成后再引用迭代块内部的变量

  

### 配置迭代块中的标题解析

输入变量：

变量1：`index`，使用`/`引用迭代搜索的`index`（即迭代器的索引）

变量2：`item`，使用`/`引用迭代搜索的`/item`（即当前迭代的条目对象）

  

编写python代码：

```
def main(index: int, item) -> dict:
    return {
        "title": str(item["title"])
    }
```

  

输出变量：`title`，类型`String`

  

### 配置迭代块输出变量

配置迭代块的输出变量，使用`/`引用迭代块内部的`title`变量。

  

## 配置结束节点

迭代块的下一步增加`结束`节点。

输出变量：使用`/`引用迭代搜索的`output`变量
