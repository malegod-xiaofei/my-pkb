# 04 大模型基础面试题

> 参考笔记：[[AI大模型学习路线图]]、[[大模型应用路线图]]、[[8 预训练模型]]、[[10 Hugging Face生态]]、[[Ollama使用]]、[[01-企业级大模型的部署]]。

## 1. 什么是大语言模型？

**答案要点：**

大语言模型是基于海量文本数据和大规模参数训练的语言模型，通常以 Transformer 为基础架构，具备文本理解、生成、推理、问答、代码生成、多轮对话等能力。

大模型与传统 NLP 模型的重要区别：

- 参数规模更大。
- 训练数据更广。
- 具备上下文学习、指令跟随和一定泛化能力。
- 可以通过 Prompt、RAG、工具调用等方式完成多类任务。

## 2. 大模型通常经历哪些训练阶段？

**答案要点：**

常见阶段包括：

1. 预训练：在大规模语料上学习语言规律和通用知识。
2. 指令微调：使用指令-回答数据训练模型理解人类指令。
3. 对齐训练：让模型输出更符合人类偏好和安全要求，如 RLHF、DPO。
4. 领域微调：针对垂直领域数据进一步适配，例如医疗、法律、金融、企业知识。

## 3. 预训练、微调和 RAG 有什么区别？

**答案要点：**

- 预训练：从海量通用数据中学习基础语言能力，成本最高。
- 微调：在已有模型基础上用特定数据更新参数，让模型适配任务或领域。
- RAG：不改模型参数，通过检索外部知识增强回答。

对比：

- 知识频繁更新或私有知识问答：优先 RAG。
- 需要改变输出风格、格式、任务习惯：可考虑微调。
- 从零训练大模型：一般只有大厂或研究机构具备条件。

## 4. 什么是指令微调？

**答案要点：**

指令微调是使用“指令 + 输入 + 回答”格式的数据继续训练模型，使模型更好理解用户需求并按照指令输出。

作用：

- 提升指令跟随能力。
- 改善多任务泛化能力。
- 让模型回答更符合人类使用习惯。

示例数据格式：

```text
指令：请总结下面这段话。
输入：......
回答：......
```

## 5. RLHF 和 DPO 有什么区别？

**答案要点：**

RLHF：

- 收集人类偏好数据。
- 训练奖励模型。
- 使用强化学习优化语言模型。
- 流程复杂，训练成本较高。

DPO：

- 直接利用偏好数据优化模型。
- 不显式训练奖励模型。
- 实现更简单，训练更稳定。

面试中可以总结：二者都用于模型对齐，DPO 是更直接、更简化的偏好优化方法。

## 6. 什么是大模型幻觉？为什么会产生？

**答案要点：**

幻觉是指模型生成看似合理但事实错误、无依据或编造的内容。

原因包括：

- 模型本质上是基于概率生成下一个 token。
- 训练数据中没有相关知识或知识过时。
- Prompt 信息不足或有歧义。
- 检索内容不准确。
- 模型缺乏事实核验机制。

降低幻觉的方法：RAG、引用来源、约束 Prompt、拒答机制、工具核验、人工审核、评测监控。

## 7. 什么是上下文窗口？长上下文是否一定更好？

**答案要点：**

上下文窗口是模型一次请求中可处理的最大 token 数，包含系统提示词、用户输入、历史对话、检索内容和模型输出。

长上下文的优势：

- 能输入更多背景资料。
- 支持长文档问答和复杂任务。

但不一定越长越好：

- 成本更高。
- 延迟更大。
- 模型可能忽略中间信息。
- 噪声过多会影响回答质量。

工程中通常结合 RAG、摘要、重排和上下文压缩使用。

## 8. Temperature、Top-K、Top-P 分别控制什么？

**答案要点：**

- Temperature：控制随机性。值越低越稳定，值越高越发散。
- Top-K：只从概率最高的 K 个 token 中采样。
- Top-P：从累计概率达到 P 的候选 token 中采样。

常见设置：

- 事实问答、代码生成、信息抽取：较低 temperature。
- 创意写作、头脑风暴：较高 temperature。

## 9. 什么是 Token？为什么大模型按 Token 计费？

**答案要点：**

Token 是模型处理文本的基本单位，可以是一个字、一个词、一个子词或符号片段。

大模型计算成本与 token 数强相关：

- 输入 token 越多，注意力计算和 KV Cache 占用越大。
- 输出 token 越多，生成时间和成本越高。

因此 API 服务通常按输入 token 和输出 token 分别计费。

## 10. 什么是 LoRA？为什么它适合大模型微调？

**答案要点：**

LoRA 即 Low-Rank Adaptation，核心思想是冻结原始模型参数，只训练额外的低秩矩阵。

优势：

- 显著减少训练参数量。
- 降低显存和计算成本。
- 多个 LoRA 权重可以灵活切换。
- 适合领域适配、风格适配和小规模任务微调。

## 11. QLoRA 是什么？

**答案要点：**

QLoRA 是在量化模型基础上进行 LoRA 微调的方法，通常将基础模型量化到 4-bit，然后训练 LoRA 适配参数。

优势：

- 大幅降低显存占用。
- 让资源有限的环境也能微调较大模型。
- 适合实验和中小规模领域适配。

需要注意：量化可能带来一定精度损失，最终效果要通过评测验证。

## 12. 大模型量化是什么？有什么影响？

**答案要点：**

量化是将模型权重或激活从高精度格式转换为低精度格式，例如 FP16 → INT8 / INT4。

优点：

- 降低显存占用。
- 提升推理速度。
- 降低部署成本。

缺点：

- 可能造成精度下降。
- 对复杂推理、数学、代码能力影响更明显。
- 不同模型和量化方法效果差异较大。

## 13. 大模型推理阶段的主要瓶颈有哪些？

**答案要点：**

常见瓶颈：

- GPU 显存不足。
- 首 token 延迟高。
- 输出 token 生成速度慢。
- 并发请求调度困难。
- KV Cache 占用大。
- 上下文过长导致计算成本高。

优化方向：量化、KV Cache 优化、批处理、流式输出、模型蒸馏、选择更小模型、使用 vLLM 等推理框架。

## 14. 大模型应用中 Prompt、RAG、微调应该如何选择？

**答案要点：**

- Prompt：适合轻量任务调整、格式控制、角色设定，成本最低。
- RAG：适合补充私有知识、实时知识和可追溯问答。
- 微调：适合固定任务模式、垂直领域语言风格、特定输出格式或复杂行为对齐。

一般优先顺序：先 Prompt，再 RAG，最后根据效果和成本决定是否微调。

## 15. 如何评价一个大模型应用是否能上线？

**答案要点：**

需要从多维度评估：

- 效果：准确性、相关性、完整性、幻觉率。
- 性能：延迟、吞吐、并发能力。
- 成本：token 成本、模型调用成本、硬件成本。
- 安全：敏感信息、越权访问、有害内容。
- 稳定性：超时、重试、降级、监控。
- 可维护性：Prompt 版本、知识库版本、日志追踪、评测集。
